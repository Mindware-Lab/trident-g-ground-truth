# SR Far-Transfer Protocol — Updated for the C-Control Architecture

**Status:** revised alignment note  
**Purpose:** update the SR relational-space hypothesis so it remains consistent with the revised IQ Coach architecture, where the adapted MFT-M/CCC task is treated as **C-Control / Cognitive Bandwidth in bits/sec**, not as a Zone or state-classification layer.

---

## 0. Executive verdict

Yes — the SR far-transfer hypothesis still holds.

The core claim remains strong:

> Trident-G far-transfer training may expand and reshape the learner's effective SR relational space by training variable abstraction and predictive transition structure across multiple relation families. With repeated wrapper variation, boundary testing, feedback and consolidation, these SR-style maps may become sufficiently compiled that candidate inferences are generated semi-automatically, while the global workspace selectively samples, verbalises, tests and stabilises the most relevant candidates.

The main update is architectural. The new IQ Coach stack should now be written as:

```text
C-Control / Cognitive Bandwidth
→ variable abstraction
→ binding
→ relational transformation tracking
→ SR-style successor prediction
→ wrapper recovery
→ global-workspace prompt selection
→ real-world mission deployment
→ delayed re-entry
→ slow schematic Gc consolidation
```

The adapted MFT-M/CCC task is therefore not a state classifier. It is the **entry-capacity layer** that measures and trains controlled perceptual evidence extraction in bits/sec. SR relational-space training begins once that evidence stream is carved into usable variables and relations.

---

## 1. Revised SR relational-space hypothesis

The updated hypothesis is:

> Relational far-transfer training is hypothesised to expand the learner's effective SR relational space by first strengthening controlled evidence extraction, then training predictive transitions across multiple relation families and wrappers. With repetition, feedback, wrapper variation and consolidation, these SR-style maps may generate candidate inferences semi-automatically, functioning as a fast-access relational search substrate. The global workspace then does not compute all inferences from scratch; it selects, verbalises, tests and constrains the most relevant SR-generated candidates through meta-epistemic prompts and task goals. Far transfer occurs only when this SR-guided inference remains portable across wrappers, vertically integrated with attention, WM and control policies, coupled to real-world niche cues, and recoverable after delay.

This preserves the original theory while making the C-Control layer explicit.

---

## 2. Why the update is needed

The earlier version said, correctly, that SRs encode:

```text
state → transition → consequence → next reachable state
```

rather than simple:

```text
stimulus → response
```

That still holds. The refinement is that the system cannot build useful SR maps over raw perceptual surfaces. First, it must extract controlled evidence and carve the state space into usable variables.

The revised hierarchy is:

```text
raw display
→ controlled evidence extraction
→ feature variable
→ bound state
→ relation
→ transition
→ successor horizon
→ candidate action / inference
```

So C-Control is not outside the SR theory. It is the **front-end capacity condition** for SR training.

In product terms:

```text
C-Control = Cognitive Bandwidth
Relational WM = variable binding and transformation tracking
SR horizon = predictive path sampling
Mindware script = global-workspace control handle
```

## 3. What remains unchanged

The following parts of the original SR hypothesis remain valid.

### 3.1 The target is relational space, not more facts

The training aim is not simply to add knowledge, tricks or strategies. It is to increase the:

```text
dimensionality
reach
resolution
usability
portability
```

of the learner's relational state space.

Training over relation families such as:

```text
sameness / difference
order
transitivity
containment
exclusion
conjunction
probabilistic transition
path constraint
counterfactual branch
equivalence class
```

gives the learner more abstract variables and transition operators.

### 3.2 SR training operates over variables, not surfaces

This remains the most important qualification:

```text
The SR space must be trained over variables, not surfaces.
```

Multiple puzzle types, Gabor fields or optic-flow displays only help if the learner is repeatedly pushed to identify the invariant variable or relation underneath the wrapper.

A wrapper swap is useful only when it asks:

```text
What is the same relation in the new surface?
```

not merely:

```text
Can the learner perform another perceptual task?
```

### 3.3 Semi-automatic inference must remain controlled

The original wording still needs to be retained:

```text
candidate relational trajectories are pre-activated or cheaply sampled
```

The goal is not rigid stimulus-response habit. The goal is that some paths, constraints, likely consequences and possible next moves become easier for the system to sample.

The global workspace still has to select, test and constrain those candidates.

### 3.4 Consolidation and delayed re-entry are essential

Far transfer still requires more than in-session improvement.

The key test is:

```text
Can the relation be recovered after delay, under changed wrapper conditions, and then used in a different problem or mission context?
```

Without delayed re-entry, the protocol risks becoming local task learning rather than slow schematic Gc construction.

```text

→ wider relational search space

→ global-workspace selection and validation

→ delayed re-entry
```

This is the cleanest version of the Trident-G far-transfer stack.

The previous mechanism:

```text
multi-relation training
→ richer variable abstraction
→ richer SR transition maps
→ wider relational search space
→ semi-automatic candidate inference generation
→ global-workspace selection and validation
→ slow schematic Gc consolidation
```

still holds, but it should now be preceded by:

```text
C-Control / Cognitive Bandwidth

```

## 6. Relationship to the separate Zone / Flow app

The SR protocol should not depend on Zone-state classification.

IQ Coach should say:

```text
C-Control measures and trains controlled perceptual evidence throughput.
Relational WM binds variables and tracks transformations.
SR horizon training samples likely future states.
Mindware prompts help users select and test useful candidate inferences.
```

IQ Coach should not say:

```text
The C-Control task detects flow.
The SR task diagnoses readiness.
The task classifies MIND_READY / MIND_FLAT / MIND_LOCKED_IN states.
```

The separate Zone / Flow app may still use cognitive, subjective and physiological signals to support readiness, reset, activation or widening routes, but that is a different product interpretation.

The clean split is:

```text
IQ Coach
= capacity, relation, SR and transfer training

Zone / Flow
= readiness, subjective flow, state support and possible HRV/body integration
```

## 9. Updated three qualifications

The original three qualifications remain, with one addition.

### 1. C-Control before SR

The learner first needs sufficient controlled evidence extraction to carve stable variables from the display.

```text
No reliable variable extraction → no reliable SR map.
```

### 2. Variables, not surfaces

The SR space must be trained over variables and relations, not stimulus surfaces.

```text
surface variation only matters when it tests invariant recovery.
```

### 3. Semi-automatic but not rigid

Automaticity should mean cheap candidate generation, not brittle fast-Gc habit.

```text
SR maps should propose paths; the global workspace still validates them.
```

### 4. Consolidation and delayed re-entry

The far-transfer claim depends on delayed recovery.

```text
Immediate task improvement is not enough.
The relation must survive time, wrapper variation and new problem contexts.
```

## 10. Most important validation prediction

The most important prediction is:

```text
C-Control improvement should support better variable extraction,
which should support relational-WM gains,
which should support wrapper recovery,
which should mediate delayed reasoning-transfer gains.
```

A practical validation chain is:

```text
C-Control gain
→ improved feature discrimination / reduced lapse rate
→ improved State-WM and A-Bind
→ improved R-Bind and S-Horizon
→ reduced W-Recovery cost
→ improved untrained reasoning-transfer pulse
→ delayed retention
```

If users improve only on trained Gabor tasks but do not show wrapper recovery or delayed reasoning-transfer gains, the far-transfer claim should be weakened.
