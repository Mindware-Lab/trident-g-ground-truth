
# THEORETICAL MAPPING: Non-Verbal SR Training → Explicit Reasoning
## Why the Extension Makes Sense (and How to Do It Right)

---

## 1. THE CORE THEORETICAL CLAIM

The extension from non-verbal SR training to explicit symbol/semantic/nonsense 
reasoning is not just sensible — it is the central mechanism of **far transfer** 
as defined in your Trident-G protocol. But it only works if the mapping is 
**structural**, not merely correlational.

The claim is:

```
Procedural relational knowledge (SR matrix)
→ Explicit relational representation (mindware script)
→ Cross-context deployment (wrapper recovery)
→ Delayed consolidation (Gc banking)
```

This maps onto the Gf → Gc → Gf spiral:
- **Gf phase**: Non-verbal SR training discovers relational invariants through 
  near-critical search
- **Gc phase**: Explicit reasoning crystallizes those invariants into portable 
  symbolic structures
- **Re-entry**: Richer Gc modules expand future Gf search space

## 2. WHY THE EXTENSION IS THEORETICALLY SOUND

### 2.1 The Dual-Process Account

| Process | Non-Verbal SR Training | Explicit Reasoning |
|---|---|---|
| **System** | Type 1 (fast, automatic, procedural) | Type 2 (slow, deliberate, symbolic) |
| **Neural basis** | Hippocampus / basal ganglia / visual cortex | Prefrontal cortex / global workspace |
| **Representation** | Distributed SR matrix M(s,s') | Discrete symbolic rules |
| **Access** | Implicit "feel" for what comes next | Explicit "knowing" that A > B |
| **Transfer** | Within-modality, near-transfer | Cross-domain, far-transfer |
| **Limitation** | Cannot verbalize or deploy in new contexts | Effortful, capacity-limited |

The two systems are **complementary**, not redundant:
- **SR training** builds the *substrate* — the predictive map that makes certain 
  transitions feel "natural"
- **Explicit reasoning** builds the *handle* — the symbolic representation that 
  allows the relation to be consciously selected, manipulated and deployed

### 2.2 The Mindware Script Hypothesis

Stanovich's "mindware" concept (adapted in your spec) posits that explicit 
reasoning tools are like software installed on hardware. The SR matrix is the 
hardware; explicit rules are the software.

```
SR matrix (hardware)
    ↓ (becomes available to global workspace)
Explicit rule (software/mindware)
    ↓ (can be selected, combined, deployed)
Reasoning operation (computation)
```

Without the SR substrate, explicit rules are **inert** — the user can state 
"A > B, B > C, therefore A > C" but cannot *feel* why it must be true.
Without explicit rules, the SR substrate is **inaccessible** — the user can 
*feel* that a transition is wrong but cannot explain or generalize it.

### 2.3 The Global Workspace Bridge

Dehaene's Global Neuronal Workspace (GNW) theory provides the mechanistic bridge:

```
Hippocampal SR map (implicit predictive landscape)
    ↓ (selected relation enters GNW)
Global workspace (relation becomes consciously available)
    ↓ (broadcast to prefrontal systems)
Explicit symbolic representation (mindware script)
    ↓ (can be verbalized, combined, stored)
Reasoning operation (conscious inference)
```

The critical insight: **GNW selection requires the relation to be sufficiently 
strong and stable** to win the "neural popularity contest." SR training 
strengthens the relation to the point where it can enter GNW. Explicit 
reasoning training then gives it a symbolic handle.

### 2.4 The Transfer Hierarchy

Your Trident-G protocol defines three transfer axes:

| Transfer Type | Non-Verbal SR | Explicit Reasoning | Combined |
|---|---|---|---|
| **Horizontal** | Same relation, different visual surfaces | Same relation, different verbal wrappers | Same relation, ANY surface |
| **Vertical** | Signal → variable → relation | Relation → inference → policy | Full stack |
| **Niche** | Visual prediction | Real-world decision | Cue-linked action |

The extension from non-verbal to explicit is **vertical transfer within the 
horizontal axis** — the same relation (trained visually) is recovered in 
symbolic/verbal form.

## 3. THE MAPPING: Four SR Classes → Four Reasoning Classes

### 3.1 CHAIN FLOW → Order/Chain Reasoning

**SR Training (Non-Verbal)**:
- User sees Gabor stream: position + orientation advance together
- Response: Continue/Break (motoric)
- Learning: Procedural sense of "this follows" vs. "this breaks"
- Neural basis: Hippocampal sequence prediction (CA3→CA1)

**Explicit Reasoning Bridge**:
- Symbolic: "A > B, B > C, therefore A > C"
- Real-language: "Mira is more experienced than Leo..."
- Nonsense: "The flarn is taller than the nidge..."

**The Mapping**:
```
Visual chain: [pos1/ori0°] → [pos2/ori45°] → [pos3/ori90°]
    ↓ (abstraction)
Relational invariant: "advance by +1 step in both dimensions"
    ↓ (symbolization)
Symbolic rule: "If A > B and B > C, then A > C"
    ↓ (generalization)
Cross-wrapper: Works for any ordered dimension
```

**Why it works**: The SR matrix has already encoded the transitive structure. 
The explicit rule simply provides a symbolic handle for the same operation. 
The user "recognizes" transitivity because their hippocampus has already 
learned it.

**Critical lure**: Indeterminate chain (A > B, C > B, therefore A ? C). The 
SR-trained user should *feel* the gap — the prediction doesn't activate — 
which maps to explicit "Cannot tell."

---

### 3.2 CHANGE FLOW → Transformation/Analogy Reasoning

**SR Training (Non-Verbal)**:
- User sees pairs: A→B, X→Y
- Response: Same/Different transformation (motoric)
- Learning: Procedural sense of "same kind of change"
- Neural basis: Hippocampal transformation operator T(s) = s'

**Explicit Reasoning Bridge**:
- Symbolic: "A→B = +2, C→D = +2, therefore same relation"
- Real-language: "Free plan→premium by adding support; basic→premium by adding setup"
- Nonsense: "Flarn→nidge by adding glorp; borp→velk by adding glorp"

**The Mapping**:
```
Visual transformation: 0°→45° (rotation +45°)
    ↓ (abstraction)
Relational invariant: "rotation by +45° operator"
    ↓ (symbolization)
Symbolic rule: "A:B :: C:D if transformation is identical"
    ↓ (generalization)
Cross-wrapper: Same operator, any operands
```

**Why it works**: The SR matrix has cached the transformation operator across 
multiple starting states. The explicit analogy rule provides a symbolic handle 
for "same operator, different operands."

**Critical lure**: Same endpoint, different transformation (0°→45° vs. 90°→45°). 
The SR-trained user should *feel* the difference in "direction" even though 
the result is the same.

---

### 3.3 RULE FLOW → Rule/Constraint Reasoning

**SR Training (Non-Verbal)**:
- User sees transitions: some smooth (valid), some jarring (invalid)
- Response: Valid/Invalid (motoric)
- Learning: Procedural sense of "this is allowed" vs. "this is blocked"
- Neural basis: Hippocampal feasibility structure (zero vs. non-zero probability)

**Explicit Reasoning Bridge**:
- Symbolic: "A AND B → C; A present, B present, therefore C"
- Real-language: "Lead is high priority only if budget AND urgent need..."
- Nonsense: "Zindle is norp only if blue AND hollow..."

**The Mapping**:
```
Visual rule: Can only move to adjacent positions
    ↓ (abstraction)
Relational invariant: "adjacency constraint"
    ↓ (symbolization)
Symbolic rule: "IF adjacent(position) THEN valid"
    ↓ (generalization)
Cross-wrapper: Any constraint system
```

**Why it works**: The SR matrix has encoded which transitions have zero 
probability (blocked). The explicit rule provides a symbolic handle for the 
constraint boundary.

**Critical lure**: Missing condition (A present, B absent, therefore C). The 
SR-trained user should *feel* that something is "missing" — the prediction 
doesn't fully activate — which maps to explicit "Invalid."

---

### 3.4 PATH FLOW → Prediction/Strategic Path Reasoning

**SR Training (Non-Verual)**:
- User sees current state + two candidates
- Response: Select more likely/reachable (motoric)
- Learning: Procedural sense of "this way feels right"
- Neural basis: Hippocampal predictive map (discounted future occupancy)

**Explicit Reasoning Bridge**:
- Symbolic: "P(B|A) high, A occurred, therefore B likely"
- Real-language: "Demo requests predict conversion; customer asked technical question..."
- Nonsense: "Zork predicts glorp; flarn asked zork..."

**The Mapping**:
```
Visual path: From pos2/ori45°, pos3/ori90° is likely, pos1/ori0° is less likely
    ↓ (abstraction)
Relational invariant: "reachability gradient from current state"
    ↓ (symbolization)
Symbolic rule: "If path through B reaches target, and path through C does not, 
                then B is more likely"
    ↓ (generalization)
Cross-wrapper: Any predictive system
```

**Why it works**: The SR matrix has cached the discounted sum of future 
occupancies. The explicit rule provides a symbolic handle for "what is 
reachable/likely from here."

**Critical lure**: Rare treated as impossible. The SR-trained user should 
*feel* that a path is still "there" even if rarely used — the prediction 
activates weakly but not zero.

## 5. WHY THIS IS NOT JUST "DOING TWO THINGS"

### 5.1 The Complementarity Argument

Non-verbal SR training and explicit reasoning training are not redundant. They 
are **complementary layers of the same architecture**:

| Layer | Function | Training |
|---|---|---|
| **SR matrix** | Implicit predictive map | Non-verbal Gabor stream |
| **Variable abstraction** | What feature matters | Implicit through prediction error |
| **Relational invariant** | What survives wrapper change | Tested through wrapper swap |
| **Explicit rule** | Symbolic handle for relation | Component 4 reasoning items |
| **Mindware script** | Reusable reasoning tool | Component 4 practice |
| **Global workspace** | Conscious selection | Meta-epistemic training |
| **Niche deployment** | Real-world action | Implementation intentions |

Each layer depends on the one below it. You cannot build explicit reasoning 
on a weak SR substrate — the relation won't be strong enough to enter global 
workspace. You cannot build niche deployment without explicit rules — the user 
won't know which tool to select.

### 5.2 The "Forward Flow" Argument

Your goal of "forward flow" requires both systems:

```
Gabor stream (SR training)
    ↓ (automatic prediction)
"This feels right" (procedural knowledge)
    ↓ (enters global workspace)
"This is transitivity" (explicit recognition)
    ↓ (mindware script selected)
"Therefore A > C" (reasoning operation)
    ↓ (automated through practice)
"This feels right" (re-automatized explicit rule)
```

The explicit rule, once practiced enough, becomes **proceduralized** itself — 
it becomes a new Type 1 process. This is the "forward flow": explicit 
knowledge that has been practiced until it becomes automatic.

### 5.3 The Empirical Argument

The literature supports this complementary approach:

- **Implicit statistical learning** (Pavlidou et al., 2019): Visual sequence 
  learning transfers to explicit reasoning tasks citeweb_search:8#2
- **Relational integration training** (Wang et al., 2025): Training relational 
  integration improves fluid intelligence through frontoparietal modulation citeweb_search:3#0
- **Inductive reasoning training** (Klauer & Phye, 2008): Explicit relational 
  reasoning training shows promising transfer effects
- **Working memory training** (Jaeggi et al., 2008): Near-transfer effects are 
  stronger when combined with strategy instruction

The pattern: **implicit training builds substrate, explicit training builds 
handles, combined training builds transfer.**

## 6. THE RISK: VERBAL CONTAMINATION

### 6.1 The Danger

If explicit reasoning training is done poorly, it can **contaminate** the SR 
substrate by:
- Encouraging verbal rehearsal instead of visual prediction
- Building wrapper-specific rules instead of abstract relations
- Creating "brittle" explicit knowledge that doesn't connect to implicit substrate

### 6.2 The Safeguards

Your spec already includes critical safeguards:

1. **Wrapper order**: Start with symbolic (pure form), then real-language, then 
   nonsense. This prevents belief-bias contamination.

2. **Lure taxonomy**: Believable-invalid lures test whether the user is using 
   the relation or just plausibility.

3. **Cannot-tell responses**: Force the user to recognize indeterminacy, not 
   just guess.

4. **Brief blocks**: 3-5 minutes prevents verbal rehearsal strategies.

5. **No long explanations**: "Show why" is optional, not forced.

6. **Same-day bridge**: Explicit block immediately follows SR block while the 
   relation is still active.

### 6.3 The Design Principle

The explicit layer should **reveal** what the SR layer has already learned, 
not **replace** it. The user should feel: "Oh, *that's* what I was doing!" 
— an "aha" moment of explicit recognition, not a new lesson.

## 8. PRACTICAL IMPLEMENTATION RULES

### 8.1 The Bridge Block Protocol

```
SESSION STRUCTURE
├── Block 1: Non-verbal SR training (15 min)
│   └── Gabor stream, Chain Flow, visual wrapper
├── Block 2: Same-day bridge (5 min)
│   ├── 2-3 explicit reasoning items (symbolic wrapper)
│   ├── Same relation family as Block 1
│   └── Brief feedback only
├── Block 3: Non-verbal SR training (15 min)
│   └── Gabor stream, Change Flow, visual wrapper
├── Block 4: Same-day bridge (5 min)
│   ├── 2-3 explicit reasoning items
│   └── Same relation family as Block 3
└── Block 5: Mixed review (10 min)
    ├── Interleaved SR + reasoning items
    └── Delayed probes from previous sessions
```

### 8.2 The Recognition Test

The bridge block should feel like a **recognition test**, not a **learning test**:

- **Wrong**: "Today we will learn about transitivity."
- **Right**: "You just practiced this relation visually. Now see if you 
  recognize it in symbols."

The user should not feel like they are learning something new. They should feel 
like they are **naming** something they already know.

### 8.3 The Scaffolding Fading

```
Week 1-2: Heavy SR training, light explicit bridge (20% of session)
Week 3-4: Balanced SR + explicit (50/50)
Week 5-6: Heavy explicit training, light SR maintenance (20% of session)
Week 7+: Mixed adaptive (based on bottleneck detection)
```

The SR training is scaffolding for explicit reasoning. Once the explicit layer 
is strong, the SR training can fade to maintenance.

## 9. SUMMARY: THE THEORETICAL ARGUMENT

### 9.1 The Claim

Extending non-verbal SR training to explicit symbol/semantic/nonsense reasoning 
is theoretically sound because:

1. **Complementarity**: The two systems (procedural SR vs. explicit symbolic) 
   serve different but complementary functions
2. **GNW bridge**: Explicit representation requires the relation to be strong 
   enough to enter global workspace — SR training provides this strength
3. **Mindware installation**: Explicit rules are reusable handles for relations 
   discovered through SR training
4. **Proceduralization**: Explicit rules, once practiced, become automated 
   themselves — this is the "forward flow"
5. **Transfer hierarchy**: Horizontal transfer (wrapper change) requires both 
   implicit recovery AND explicit recognition

### 9.2 The Condition

The extension only works if:

1. **Structural mapping**: The explicit items map to the same relational 
   invariant as the SR training (not just correlated content)
2. **Timing**: Explicit bridge occurs while the SR relation is still active 
   (same-day, not delayed)
3. **Recognition, not learning**: The explicit layer should feel like naming, 
   not learning
4. **Wrapper progression**: Symbolic → real-language → nonsense (prevents 
   belief-bias contamination)
5. **Lure resistance**: Explicit items must include lures that test whether 
   the user is using the relation or just plausibility

### 9.3 The Prediction

If the mapping is done correctly, you should see:

```
Strong SR training (Component 3)
    ↓
Faster explicit reasoning acquisition (Component 4)
    ↓
Higher wrapper recovery (R-WRecovery)
    ↓
Better delayed transfer (R-Delay)
    ↓
Improved niche deployment (real-world reasoning)
```

The SR training is not a prerequisite in the sense of "must complete before 
starting explicit." It is a **substrate** that makes explicit training more 
efficient, more transferable, and more durable.

```
Visual prediction (SR training)
    ↓
Explicit recognition (bridge block)
    ↓
Symbolic manipulation (reasoning training)
    ↓
Re-automatization (practice)
    ↓
Fluid reasoning (forward flow)
```

The non-verbal SR layer discovers relational invariants. The explicit reasoning 
layer crystallizes them into portable mindware. The combination produces the 
"forward flow" workspace you are aiming for — where reasoning feels automatic 
because it is grounded in a rich predictive substrate.

The key is **structural mapping** (same relation, different surface) and 
**recognition timing** (explicit bridge while SR is active). Done right, the 
user experiences explicit reasoning not as a new skill to learn, but as a 
language for describing what they already know.
