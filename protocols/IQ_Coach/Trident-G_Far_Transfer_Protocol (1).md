## Trident-G Far Transfer Protocol

The Trident-G protocol trains far transfer by combining a **vertical transfer cycle**, a **horizontal transfer cycle**, **niche deployment** and **delayed consolidation**.

The **vertical transfer cycle** trains the control chain: state readiness, cognitive-control bandwidth, relational working memory, SR-style inference, mindware selection, global-workspace checking, action and feedback.

The **horizontal transfer cycle** trains invariant recovery: the same underlying cognitive operation must survive near-miss interference, changes in surface features, modality, wrapper, context and delay.

Together, these cycles produce a **Gf → Gc → Gf spiral**. Fluid cognition searches the Griffiths/Ψ-band, successful structures are stabilised into modular Gc, and later Gf re-enters the Ψ-band with richer predictive material to recombine.

## 1. Vertical transfer cycle

The vertical transfer cycle asks whether the same adaptive policy can be reconstructed across the layers of cognition and action.

| Layer                                                 | Protocol function                                                                                                                                                        | Core question                                                                                                     | Main app component                            |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| **0. Zone Coach / Zone-Cog readiness gate**                          | Estimates task-active control, engagement-vigilance, personal deviation and uncertainty to route the next training block                                                                                                | “Am I ready for this kind of cognitive load now?”                                                                 | Zone Coach / Zone-HRV                         |
| **1. CCC / cognitive-control bandwidth**              | Trains reliable control under increasing information-rate demand                                                                                                         | “How much uncertainty can I coordinate without overload or brittle collapse?”                                     | adapted MFT-M / CCC                           |
| **2. Relational WMC + SR generative-space expansion** | Trains the active workspace for holding variables while expanding the partly automated predictive space of possible relations, transitions, consequences and next states | “Can I hold the relevant variables online, update their relations, and generate useful next-state possibilities?” | relational n-back / WM-G / SR-style inference |
| **3. Specific mindware scripts**                      | Provides reusable reasoning, reframing and inference procedures                                                                                                          | “Which tool fits this problem?”                                                                                   | IQ Coach / R Coach prompts                    |
| **4. Meta-epistemic / global-workspace control**      | Selects, monitors and regulates the mindware script                                                                                                                      | “Should I open, constrain, test, reframe, slow-check, reset or act?”                                              | global-workspace checking layer               |
| **5. Niche deployment / application**                 | Tests whether the policy works in real tasks, study, pain, anxiety, work, sport, etc.                                                                                    | “Can I use this in the real situation?”                                                                           | missions / implementation intentions          |
| **6. Delayed re-check / consolidation**               | Tests whether the policy survives time, wrapper change and real-world reuse                                                                                              | “Has this become portable slow Gc rather than a task trick?”                                                      | delayed probes / baseline tracking            |

### Compact vertical formulation

```text
Zone Coach gates the state.
CCC trains control bandwidth.
Relational WMC expands the active generative inference space.
Mindware supplies reusable reasoning and reframing tools.
Meta-epistemic control selects and regulates those tools.
Niche deployment tests real-world coupling.
Delayed probes test consolidation into slow schematic Gc.
```

**Step 2**  has two linked functions:

| Step 2 function                   | Meaning                                                                                                                  |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **Relational WMC**                | Hold, bind, update and compare variables under load                                                                      |
| **SR generative-space expansion** | Build a richer predictive map of what could follow, what actions change the state, and which future states are reachable |

So the progression becomes:

```text
variable holding
→ relation binding
→ transition prediction
→ next-state generation
→ portable inference structure
```

Once Step 2 expands the generative space, Steps 3–4 can decide whether to **open**, **constrain**, **test**, **reframe**, **slow-check**, **reset** or **act**.

**Steps 3–4** operate as a coupled layer. Mindware scripts provide the reusable tools: constraint checking, boundary testing, counterfactual search, appraisal reframing, SR look-ahead, expected-utility reasoning, Bayesian updating, etc. Meta-epistemic control governs when to use them: when to open the search space, when to constrain it, when to test a boundary, when to slow down, when to reset, and when to act.

## 2. Horizontal transfer cycle

Horizontal transfer prevents thin, task-specific learning. It asks whether the same cognitive operation survives near-miss interference, wrapper change, modality change, context change and delay.

For any relational target **X**, the horizontal transfer cycle is:

```text
relational target X
→ training target with near-miss foils
→ local tuning
→ asymptote detection
→ wrapper / modality / context perturbation
→ temporary dip
→ invariant recovery
→ local tuning with foils
→ delayed re-check
→ slow Gc consolidation
```

This cycle can be applied at every vertical layer. The meaning of “relational target X” changes by layer, but the horizontal logic is the same: train the invariant, sharpen it with foils, perturb the surface near asymptote, recover the invariant, retune with foils, and bank only what survives delay.

### Why near-miss foils matter

Near-miss foils are the specificity engine. They prevent overgeneralisation and brittle Gc by forcing the learner to distinguish:

```text
same-looking but different
different-looking but same
valid relation
near relation
wrong binding
wrong lag
wrong boundary
wrong script
wrong context
```

Foils are therefore not a late add-on. They are part of the training target from the beginning. They force interference control before local asymptote and again after invariant recovery.

### Why wrapper perturbation matters

Wrapper perturbation is the transfer engine. When performance begins to flatten, the surface is changed while the invariant is preserved. This disrupts surface-specific automation and reopens exploratory search.

A successful horizontal cycle should show a **dip-and-recovery** pattern:

```text
temporary dip
= surface-specific automation has been disrupted

invariant recovery
= the deeper relation has been rediscovered under changed conditions
```

So the compact horizontal formulation is:

```text
Train the target with foils.
Tune locally.
Perturb the wrapper at asymptote.
Recover the invariant.
Retune with foils.
Bank only what survives delay.
```

## 4. Combined transfer principle

The complete protocol combines the vertical transfer cycle and the horizontal transfer cycle.

```text
Vertical transfer cycle
= the policy reconstructs across layers.

Horizontal transfer cycle
= the invariant survives foils, wrapper change and delay.

Niche transfer
= the policy activates in the real environment.

Delayed consolidation
= only what survives becomes slow schematic Gc.
```

Or more compactly:

```text
same policy
→ across layers
→ across foils
→ across wrappers
→ across real-world cues
→ across time
```

The protocol does not merely ask whether the learner improves on a task. It asks whether the same adaptive policy can be reconstructed across state, attention, WM/SR inference, mindware selection, global-workspace checking, action, feedback, surface variation and delay.

## 5. Gf → Gc → Gf spiral

The combined vertical and horizontal cycles produce the **Gf → Gc → Gf spiral**.

During **Gf**, the learner searches a near-critical relational space, balancing exploratory entropy with task-relevant constraint. Near-miss foils sharpen the target by preventing over-broad generalisation. Wrapper perturbations reopen exploration when local automation begins to harden.

When a useful relation survives foil interference, wrapper variation, feedback and delay, it is partially stabilised as **Gc** within modular predictive maps. This Gc should not be a narrow surface trick. It should be a bounded, portable relational structure: a partly automated successor-representation module, control policy or mindware script.

Later, that Gc structure becomes material for a new round of **Gf**. Because hierarchical modular organisation allows local modules to stabilise while remaining weakly coupled to wider networks, fluid cognition can re-enter the Griffiths/Ψ-band and recombine these richer modules into new generative possibilities.

So the final spiral is:

```text
Gf searches the Griffiths zone.
The horizontal cycle isolates the invariant.
The vertical cycle binds the control chain.
Gc stabilises the invariant into modular predictive structure.
Later Gf recombines those modules into a richer generative space.
```

In one sentence:

**Far transfer is trained by repeatedly cycling useful relations through near-miss foils, local tuning, wrapper perturbation, vertical policy reconstruction, niche deployment and delayed consolidation, so that what becomes Gc is not a surface routine but a modular relational structure that later expands the generative space for new Gf inference.**

## Zone Coach: Cognitive readiness layer: task-active control and engagement-vigilance

Before the vertical and horizontal transfer cycles begin, the protocol can use a brief cognitive readiness gate. This gate estimates whether the person is currently in a suitable state for high-load training, wrapper perturbation, problem solving or recovery.

The cognitive readiness gate has two separable outputs:

```text
task-active control profile
+ engagement-vigilance
```

Task-active control is estimated from conflict-task performance, such as Stroop and Flanker. It describes how effectively cognitive control operates once the person is engaged in the task:

| Profile                 | Meaning                                             | Training route                                                                  |
| ----------------------- | --------------------------------------------------- | ------------------------------------------------------------------------------- |
| **Regulated**           | accurate, relatively fast and resilient to conflict | proceed, increase challenge, use full vertical/horizontal transfer cycle        |
| **Slow compensatory**   | controlled but effortful, cautious or costly        | keep structure, reduce unnecessary checking, time-box, gradually increase speed |
| **Globally overloaded** | slow, inaccurate and conflict-vulnerable            | simplify, scaffold, reduce information density, delay harder perturbations      |
| **Fast brittle**        | fast response policy with elevated accuracy failure | slow-check, add disconfirmation, use checklists, train reset after errors       |

Engagement-vigilance is estimated separately, for example with SART. It reflects sustained response readiness, lapse-proneness and response-time stability. It should be treated as a continuous dimension, not as a pure arousal measure.

```text
engagement-vigilance
= preserved <----------------------------> lapse-prone
```

The practical model is therefore not one hard class, but:

```text
four task-active control probabilities
+ continuous engagement-vigilance
+ personal-baseline deviation
+ uncertainty / abstention
```

This matters for training. A learner should not be pushed into a hard wrapper perturbation merely because a task score is acceptable. If engagement-vigilance is low, the protocol may begin with activation, recovery, an easier entry task or a lower-load version of the same training target. If control is fast brittle, the next block should include slow-checking and disconfirming foils. If control is overloaded, the next block should simplify before increasing entropy. If control is regulated, the learner can proceed into the full vertical and horizontal transfer cycle.
