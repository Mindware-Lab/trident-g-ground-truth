# H-AGI (Human-AI General Intelligence) Protocol™

### Quick-reference card

```text
Regulate. Frame. Expand. Reason. Challenge. Probe. Act. Update. Re-check. Consolidate.
```

Ultra-short session version:
```text
Think first.
Use AI to expand.
Reason to filter.
Probe reality.
Learn what survives.
```

---

### Core definition

H-AGI stands for Human-AI General Intelligence.

H-AGI does not mean autonomous Artificial General Intelligence. It means Human-AI General Intelligence: a human-governed protocol for using AI as a general-purpose cognitive workspace to augment transferable human intelligence.

The H-AGI Protocoll™ is not a prompt library. It is a training protocol for AI-era intelligence. Its purpose is to help users frame problems, reason relationally, challenge assumptions, generate alternatives, design causal probes, make decisions, act in the world, update from feedback and consolidate reusable intelligence.

The protocol has three linked objectives:

```text
1. Augment intelligence through AI-supported extended cognition.
2. Protect intelligence by preventing passive cognitive offloading.
3. Train transferable intelligence through causal testing, wrapper variation and delayed re-checks.
```

AI is treated as a **dynamic cognitive niche**: an external environment for modelling, simulation, critique, creativity, planning, feedback and learning. But the human remains the control layer. The trained skill is not "getting AI to produce better answers". The trained skill is using AI to enlarge the search space while preserving human-led meta-epistemic control.

# 1. Why H-AGI is needed

AI lowers the cost of producing outputs. It can generate text, images, code, summaries, plans and explanations almost instantly. This creates a new problem. When answers become cheap, the scarce cognitive advantage is no longer access to information. The scarce advantage is knowing what to ask, what to trust, what to reject, what to test, what to do next and how to learn from the result.

Unstructured AI use can easily produce passive cognitive offloading:

```text
The user asks.
AI answers.
The user accepts.
No model is built.
No assumption is tested.
No skill is consolidated.
```

In that pattern, AI may increase output while weakening the user's own problem framing, reasoning, memory, creativity and judgement. H-AGI is designed to reverse that pattern.

The H-AGI pattern is:

```text
The user frames.
AI expands.
The user extracts relations.
AI challenges or varies.
The user filters.
The world tests.
The user updates.
The method is re-checked later.
```

The aim is not to replace human intelligence with machine output. The aim is to create a human-led system in which AI increases the range, precision and speed of cognition while the user's own intelligence becomes more structured, better calibrated and more transferable.

# 2. Core Trident-G principle

The H-AGI Protocol extends the Trident-G view of intelligence as adaptive control across changing problem spaces. It preserves the same basic architecture as the Trident-G Far Transfer Protocol:

```text
state readiness
→ control bandwidth
→ relational working memory
→ SR-map expansion
→ reasoning / mindware selection
→ meta-epistemic control
→ causal probe
→ niche action
→ feedback
→ delayed consolidation
```

The distinctive H-AGI move is that AI becomes an external generative workspace inside this cycle. AI can generate candidate models, alternative frames, counterexamples, plans, examples, analogies and possible actions. But those outputs are not treated as answers. They are treated as **candidate relations** that must be aligned, filtered, tested and updated.

The compact Trident-G/H-AGI principle is:

```text
AI injects entropy.
Human reasoning aligns and filters.
The world supplies feedback.
Gc stabilises what survives.
Later Gf recombines it.
```

In practical language:

```text
AI helps you see more possibilities.
You decide which possibilities are coherent, relevant and testable.
Reality tells you which model survives.
You turn the surviving method into a reusable skill.
```

# 3. The mechanistic engine: entropy, mutual information and SR-map updating

H-AGI is based on a mechanism-aware model of cognition.

At the theory level, intelligent adaptation requires a balance between two forces:

```text
entropy
= opening the search space, generating variation, exploring alternatives

mutual-information constraint
= selecting what is relevant, coherent, predictive and useful
```

Too little entropy produces rigid, repetitive, over-convergent cognition. Too much entropy produces noise, scatter and ungrounded divergence. Adaptive intelligence requires a near-critical balance where enough variation is generated to discover new structure, but enough constraint is applied to prevent collapse into irrelevance.

In H-AGI, AI acts as an external entropy source. It can rapidly generate:

```text
alternative frames
candidate explanations
analogies
counterexamples
plans
causal stories
creative variations
decision options
implementation routes
```

But AI-generated variation is not automatically useful. It must pass through human-led alignment and mutual-information filtering.

The core mechanism is:

```text
AI-assisted entropy expansion
→ relational inference
→ alignment of candidate relations
→ MI filtering for relevance and predictive value
→ SR-map update
→ causal probe
→ feedback
→ consolidation
```

The **successor-representation map**, or SR map, is the user's predictive relational model of the situation:

```text
state
→ transition
→ consequence
→ next reachable state
```

In H-AGI, every useful AI interaction should improve this map. The user should end with a clearer sense of:

```text
Where am I?
What variables matter?
What transitions are possible?
What actions change the state?
What consequences follow?
What evidence would update the model?
What next state should I aim for?
```

The protocol therefore treats intelligence as more than output generation. Intelligence is the adaptive updating of predictive relational maps under uncertainty.

# 4. Ψ-band and body-budget readiness

In the current version, the Ψ-band gate is a structured self-routing heuristic. In a later validated implementation, Zone Coach can use cognitive-test and HRV-related data to classify state and route the H-AGI session more precisely.

The first layer of H-AGI is a readiness gate. This is important because AI can amplify the user's current cognitive state. If the user is regulated, AI can expand and sharpen cognition. If the user is overloaded, AI can increase confusion. If the user is fast-brittle, AI can accelerate premature certainty. If the user is fatigued, AI can become a stimulant for shallow productivity rather than deep thinking.

The H-AGI readiness gate asks whether the user is inside a workable **Ψ-band corridor**: a state in which cognitive control, working memory, attention, autonomic reserve and vigilance are sufficient for high-load AI-assisted cognition.

The readiness gate checks four broad dimensions:

| Check                   | Question                                                 | Why it matters                          |
| ----------------------- | -------------------------------------------------------- | --------------------------------------- |
| Cognitive load capacity | Can I hold the problem frame before asking AI?           | Prevents passive offloading             |
| Autonomic reserve       | Do I feel resourced enough for uncertainty and checking? | Prevents AI use under allostatic strain |
| Vigilance stability     | Can I monitor details, sources, assumptions and errors?  | Prevents shallow acceptance             |
| Baseline deviation      | Am I unusually wired, flat, scattered or strained today? | Detects state drift                     |

## 4.1 Operational Ψ-band gate

Before using AI for high-load work, rate each item 0–2 (0 = no, 1 = partial, 2 = yes):

```text
Cognitive control:
[ ] I can state the problem without AI (0/1/2)
[ ] I can hold the main variables and their relations in mind (0/1/2)

Engagement-vigilance:
[ ] I can sustain attention for detail-checking (0/1/2)
[ ] I am not rushing to avoid thinking (0/1/2)

Autonomic reserve:
[ ] I feel resourced, not depleted or wired (0/1/2)
[ ] My body is not signaling strain (tight chest, shallow breathing, restlessness) (0/1/2)
```

## 4.2 Routing thresholds

| Total score | Route |
|-------------|-------|
| 10–12       | Full H-AGI cycle |
| 7–9         | Scaffold or reduce entropy load |
| 0–6         | Recover or activate before AI use |

## 4.3 Veto rules (mandatory overrides)

The total score can be overridden by qualitative state conflicts:

```text
If any autonomic item = 0:
  → Maximum route = scaffold, regardless of total score.
  → Rationale: depleted body-budget cannot sustain near-critical search.

If any cognitive item = 0:
  → Maximum route = simplify, regardless of total score.
  → Rationale: inability to hold the problem frame guarantees passive offloading.

If both autonomic items = 0:
  → Route = recover.
  → Rationale: allostatic strain makes AI use cognitively expensive and error-prone.

If cognitive score ≥ 4 but autonomic score ≤ 2:
  → Route = scaffold with reduced complexity.
  → Rationale: regulated mind with depleted body = compensatory performance, not true readiness.
```

The readiness gate protects the user from AI-amplified state drift.

A later validated implementation can connect this gate to the Zone Coach app, using cognitive-test and HRV-related data to classify cognitive state and route the H-AGI session more precisely. For the present protocol, the gate can remain lightweight and conceptual.

# 5. Route logic

H-AGI adapts the AI interaction to the user's current cognitive-readiness profile.

| Readiness pattern        | H-AGI route             | Rationale                                                                           |
| ------------------------ | ----------------------- | ----------------------------------------------------------------------------------- |
| Regulated                | Full H-AGI cycle        | The user can handle entropy, foils, causal probes and transfer checks               |
| Slow compensatory        | Structured scaffold     | The user can reason, but should reduce unnecessary complexity and time-box checking |
| Overloaded               | Simplify                | The system should reduce output volume, variables and simultaneous modes            |
| Fast brittle             | Slow-check              | The system should force disconfirmation, delay commitment and reopen the SR map     |
| Low engagement-vigilance | Activate or recover     | The user should not use AI as a substitute for attention                            |
| High uncertainty         | Abstain or repeat check | The system should avoid strong routing if state classification is unclear           |
| Strained body-budget     | Reduce entropy load     | Complex foils and multi-option AI outputs may overload the system                   |

The route rule is:

```text
Do not let AI amplify the wrong state.
```

Examples:

```text
Overloaded user:
Do not give ten strategic options.
Give one clear frame, one assumption check and one safe next probe.

Fast-brittle user:
Do not give persuasive fluent output quickly.
Require counterevidence, relation extraction and delayed action.

Low-engagement user:
Do not move straight into complex reasoning.
Use activation, a short human-first task or recovery.

Regulated user:
Use full AI expansion, foils, causal testing and delayed re-checks.
```

# 6. Relational inference as the core Gf mechanism

The central cognitive operation in H-AGI is **relational inference**.

Relational inference is what turns AI-generated variation into structured intelligence. It extracts, compares, aligns, tests and updates relations. It is the bridge between entropy and mutual-information constraint.

The mechanism is:

```text
local entropy
→ candidate relations
→ relational alignment
→ MI filtering
→ SR-map update
```

AI may generate the candidates, but the human must perform or supervise the relational inference.

The key question is not:

```text
What did AI say?
```

It is:

```text
What relation is AI proposing?
Is that relation valid?
What assumptions does it require?
What would make it false?
Does it improve the SR map?
Can it be tested?
```

Relational inference is the engine of fluid intelligence in the protocol. The H-AGI thinking modes are not separate from reasoning. They are applications of relational inference to different cognitive demands.

| Mode       | Relational inference operation                          | SR-map function                                   |
| ---------- | ------------------------------------------------------- | ------------------------------------------------- |
| Comprehend | Structure induction                                     | Build nodes, variables, relations and constraints |
| Challenge  | Disconfirmation search                                  | Find relations that break the current map         |
| Reframe    | Map reconfiguration                                     | Replace the relational skeleton                   |
| Probe      | Counterfactual inference                                | Test whether a transition predicts reality        |
| Decide     | Comparative evaluation                                  | Choose a path through the map                     |
| Plan       | Path search                                             | Sequence actions and checkpoints                  |
| Create     | Divergent relation generation plus convergent selection | Expand and prune the map                          |
| Learn      | Abstraction                                             | Extract invariant relations into reusable Gc      |

This means H-AGI is not a collection of "AI modes" in the shallow sense. It is a system for selecting the correct relational operation for the task.

# 7. Working memory and cognitive control

Working memory is the capacity limit on H-AGI reasoning.

In this protocol, working memory is not merely "holding information in mind". It is the active workspace for holding candidate relations while they are aligned, transformed, filtered and updated.

| Working-memory function | H-AGI meaning                                        |
| ----------------------- | ---------------------------------------------------- |
| Storage                 | Holding candidate relations in the active workspace  |
| Manipulation            | Transforming and comparing those relations           |
| Updating                | Replacing weak relations with better ones            |
| Inhibition              | Suppressing irrelevant or fluent-but-wrong relations |
| Binding                 | Connecting variables, causes, actions and outcomes   |
| Sequencing              | Holding state → transition → consequence chains      |

Cognitive control protects this workspace. It prevents the user from being captured by AI fluency, excessive options, emotional salience, social agreement or premature closure.

The control question is:

```text
Can I stay in charge of the relation space?
```

If the user cannot hold the problem frame, extract the relations or check assumptions, the protocol should reduce AI output and increase scaffolding.

# 8. The H-AGI vertical cycle (theoretical architecture)

The vertical cycle reconstructs the same adaptive policy across layers: state, attention, relational workspace, reasoning, meta-control, causal testing, action and delayed reuse.

| Layer                              | Function                                                        | Core question                                               | Example component                                                  |
| ---------------------------------- | --------------------------------------------------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------ |
| 0. Ψ-band readiness gate           | Checks cognitive/autonomic readiness                            | Am I in the right state to use AI well?                     | Self-check now, Zone Coach later                                   |
| 1. Attention and control bandwidth | Prevents passive answer consumption                             | Can I stay in control of the interaction?                   | Timed AI review, output compression, source triage                 |
| 2. External relational workspace   | Holds variables, assumptions, options and consequences          | Can I see the structure of the problem?                     | Map, matrix, dashboard, argument graph                             |
| 2.5. Relational inference engine   | Aligns and filters candidate relations                          | What relations are being proposed, and do they hold?        | Relation extraction, analogy mapping, counterfactual check         |
| 3. Mindware scripts                | Applies reusable reasoning operations                           | Which thinking mode fits this problem?                      | Comprehend, Challenge, Decide, Plan, Probe, Reframe, Create, Learn |
| 4. Meta-epistemic controller       | Regulates when to open, constrain, verify, reframe, test or act | What should I do with this uncertainty?                     | Mode router, critique gate, confidence gate                        |
| 4.5. SR-map update                 | States what changed in the predictive model                     | What state → transition → consequence relation has changed? | Model revision note                                                |
| 5. Causal probe / niche deployment | Tests the updated map in the world                              | What is the smallest safe test?                             | A/B test, prototype, decision memo, feedback tracker               |
| 6. Delayed re-check                | Tests independence from the scaffold                            | Can I recover the method later, under changed conditions?   | No-AI recall, changed-wrapper re-test                              |

Compact formulation:

```text
Readiness gates the state.
Attention protects the workspace.
AI externalises the relation space.
Reasoning aligns and filters candidate relations.
Mindware scripts guide the operation.
Meta-control selects the next cognitive move.
The SR map is updated explicitly.
Causal probes test the map.
Delayed re-checks test transfer.
```

# 9. The H-AGI horizontal cycle

The horizontal cycle prevents tool-dependent learning. It asks whether the same cognitive operation can survive changes in surface form, model, artefact, context and delay.

For any H-AGI target X:

```text
relational target X
→ AI-assisted practice with near-miss foils
→ local tuning
→ surface fluency begins to form
→ wrapper variation
→ temporary dip
→ invariant recovery
→ retuning with foils
→ causal probe
→ delayed re-check
→ Gc consolidation
```

The wrapper may change across:

```text
custom GPT
→ plain chat
→ Claude artefact
→ spreadsheet
→ diagram
→ notebook
→ no-AI scratchpad
→ real-world decision
→ next-day re-check
```

The target is not skill with one prompt. The target is recovery of the same thinking operation across changed wrappers.

Example:

```text
Target:
Detect hidden assumptions in a plan.

Initial wrapper:
AI produces a plan and asks the user to identify assumptions.

Foils:
Some assumptions are explicit, some hidden, some irrelevant.

Wrapper variation:
The same plan is shown as a table, then a memo, then a diagram, then a real decision.

Delayed re-check:
The user must identify hidden assumptions the next day without the same AI scaffold.

Consolidation:
The user develops a reusable assumption-detection script.
```

The horizontal cycle has a characteristic dip-and-recovery pattern:

```text
temporary dip
= surface-specific automation has been disrupted

invariant recovery
= the deeper relational operation has been recovered
```

Only what survives variation and delay should be treated as transferable learning.

# 10. Near-miss foils as controlled entropy injections

Near-miss foils are not just tests. They are controlled entropy injections that sharpen the user's MI filter.

They prevent AI fluency from being mistaken for truth and prevent the user from learning shallow prompt-specific routines.

| Foil type                  | What it injects                     | What it trains                       |
| -------------------------- | ----------------------------------- | ------------------------------------ |
| Plausible-but-wrong claim  | Fluent but false relation           | Fluency discrimination               |
| Missing-assumption foil    | Hidden premise                      | Assumption detection                 |
| Wrong-frame foil           | Alternative but incorrect map       | Problem-frame checking               |
| Over-broad conclusion foil | Valid relation beyond its boundary  | Generalisation control               |
| Narrow-surface foil        | Same operation in changed wrapper   | Transfer recovery                    |
| Sycophancy foil            | Agreement without challenge         | Resistance to social/AI confirmation |
| Creativity cliché foil     | Polished but low-novelty output     | Detection of pseudo-creativity       |
| False-source foil          | Unsupported evidence claim          | Source discipline                    |
| Premature-action foil      | Action before uncertainty reduction | Probe-first discipline               |
| Vague-divergence foil      | Many options with no relevance      | MI filtering                         |

Causal foils add an additional layer:

| Causal foil                 | What it prevents                                   | Example                                           |
| --------------------------- | -------------------------------------------------- | ------------------------------------------------- |
| Correlation foil            | Mistaking association for cause                    | Sales rose after an email, so the email caused it |
| Confound foil               | Missing a third variable                           | Price, timing or traffic changed simultaneously   |
| Proxy-metric foil           | Measuring the easy thing                           | Clicks improve but qualified leads fall           |
| No-baseline foil            | Interpreting change without comparison             | No before/after or control condition              |
| Outcome-switching foil      | Changing success criteria after the result         | Claiming success on a different metric            |
| Too-large intervention foil | Testing too many changes at once                   | New price, page and email together                |
| Confirmation-test foil      | Only seeking supportive evidence                   | Asking users why they liked the product           |
| AI-plausibility foil        | Accepting a coherent causal story without evidence | Smooth explanation, no discriminating test        |

The training question is:

```text
Is this relation supported by the structure of the problem, or only by the fluency of the presentation?
```

# 11. Alignment and MI-filter procedure

The alignment and mutual-information filter is the core procedure that prevents AI-generated entropy from becoming noise.

After AI produces candidate relations, reframes, claims or plans, the user runs a structured filter.

## 11.1 No-relation protocol

If the AI output contains no extractable relation — only fluent description, narrative or advice without a clear state→transition→consequence structure:

```text
→ Do not proceed to MI filter.
→ Route to Comprehend mode.
→ Ask AI to restate in state→transition→consequence format.
→ Do not proceed until a relation is explicit.
```

This prevents the filter from being applied to non-relational output.

## 11.2 MI-filter checklist

For each candidate relation, ask:

```text
1. Coherence
Does this relation fit what I already know?
If not, is the relation wrong, or is my existing map incomplete?

2. Relevance
Does this change the problem structure, or does it merely add detail?

3. Predictivity
Does this relation generate a prediction I did not already have?

4. Usefulness
If true, would this change what I should do?

5. Boundary
Where does this relation hold?
Where might it fail?

6. Evidence
What supports it?
Is the support independent of AI fluency?

7. Probe
What small test would strengthen or weaken this relation?
```

## 11.3 Classification

Classify each candidate relation as:

```text
retain    = install into the SR map
revise    = modify before installing
test      = promising but unverified; design a probe
delay     = interesting but not actionable now; schedule re-check
reject    = fails coherence, relevance or evidence checks
```

The goal is not to accept the most fluent AI output. The goal is to keep only the relations that improve the predictive map or produce a useful test.

# 12. SR-map update template

The SR-map update step makes cognitive restructuring visible. The user explicitly states what changed in their predictive model.

Use this template after meaningful AI-assisted reasoning:

```text
SR-map update

Before:
State:
What I thought the current situation was.

Transition:
What I thought would happen next, or what action I thought would move the system.

Consequence:
What outcome I expected.

Next state:
Where I thought this would lead.

After AI interaction + human reasoning:
Revised state:
Did the problem definition change?

Revised transition:
Did the causal path, action path or inference path change?

Revised consequence:
Did the predicted outcome change?

Revised next state:
Did the reachable goal or likely future state change?

Confidence:
High / medium / low, and why.

Test needed:
What would confirm, weaken or refine this update?
```

A useful SR-map update is specific. It should not say:

```text
I understand this better now.
```

It should say:

```text
I previously thought action A would lead to outcome B.
The revised map suggests action A only works if condition C is present.
The next test is to check whether condition C is actually present.
```

This turns an AI-assisted "insight" into an inspectable model change.

## 12.1 Null update protocol

If no change occurred:

```text
1. State why: "All candidate relations failed the MI filter because..."
2. State what was tested: "I now know that relation X does NOT hold under conditions Y."
3. State the implication: "This narrows the map rather than expanding it."
4. Route: Challenge or Probe mode to test the boundary of the negative finding.
```

A null update is still a valid SR-map update (constraint tightening). It should be recognized as such.

# 13. The eight H-AGI thinking modes

The eight thinking modes are practical interfaces for applying relational inference.

## 13.1 Comprehend

Purpose:

```text
Build the relational map.
```

Core questions:

```text
What are the variables?
What are the constraints?
What depends on what?
What are the states, transitions and consequences?
What is still unknown?
```

AI role:

```text
Summarise, structure, map, define terms, extract variables, identify assumptions.
```

Human role:

```text
Check the structure, reject false simplifications, identify missing relations.
```

Output:

```text
A clear SR map of the situation.
```

---

## 13.2 Challenge

Purpose:

```text
Test the map.
```

Core questions:

```text
What would make this wrong?
What assumption is weakest?
What evidence is missing?
What is the strongest counterexample?
What would a sceptic say?
```

AI role:

```text
Generate objections, alternative interpretations, weak points, edge cases and counterclaims.
```

Human role:

```text
Distinguish real counterevidence from generic critique.
```

Output:

```text
A stronger, better-bounded model.
```

---

## 13.3 Probe / Experiment

Purpose:

```text
Test the map against reality.
```

Core questions:

```text
What is the smallest safe test?
What would I expect to observe if this model is right?
What would I observe if it is wrong?
What can I test without overcommitting?
```

AI role:

```text
Suggest probes, predictions, controls, measures and possible confounds.
```

Human role:

```text
Choose the probe, act responsibly, interpret feedback.
```

Output:

```text
A discriminating test and update rule.
```

---

## 13.4 Decide

Purpose:

```text
Choose a path through the map.
```

Core questions:

```text
What are the options?
What outcomes matter?
What are the probabilities, utilities, risks and opportunity costs?
Which path has the best expected value given my aims?
```

AI role:

```text
Structure options, build decision matrices, clarify trade-offs, run sensitivity checks.
```

Human role:

```text
Set values, judge uncertainty, own the decision.
```

Output:

```text
A justified decision with explicit uncertainty.
```

---

## 13.5 Plan

Purpose:

```text
Turn a goal into state transitions.
```

Core questions:

```text
Where am I now?
What is the goal state?
What operators move me forward?
What checkpoints show progress?
What feedback loop will update the plan?
```

AI role:

```text
Break goals into steps, identify dependencies, anticipate bottlenecks, generate implementation routes.
```

Human role:

```text
Select the route that fits constraints and reality.
```

Output:

```text
A state → operator → checkpoint → feedback plan.
```

```text
Change the relational structure of the problem.
```

Core questions:

```text
Generate alternative frames, metaphors, analogies and domain transfers.
```

Human role:

```text
Filter reframes by relevance, coherence and testability.
```

```text
A revised SR map that reveals new transitions.
```

```text
Generate novel but useful relational combinations.
```

Core questions:

```text
What alternatives are not obvious?
What domains can be combined?
What constraint can be changed?
What is genuinely new rather than merely polished?
What can be tested with an audience or implementation probe?
```

AI role:

```text
Inject alternative frames, analogies, combinations and variations.
```

Human role:

```text
Generate first, filter novelty, select usefulness, test reality.
```

```text
A diverse set of candidate ideas plus a selected testable concept.
```

Purpose:

```text
Consolidate the method into portable Gc.
```

```text

Can I recover the method tomorrow?
```


```text
A reusable reasoning script or SR-map structure.
```

# 15. Active causal testing

H-AGI is incomplete without causal testing.

AI can generate explanations and plans, but intelligence is not improved until those explanations and plans are disciplined by reality.

The causal loop is:

```text
hypothesis
→ prediction
→ smallest safe intervention
→ observation
→ update
→ next action
```

The core causal question is:

```text
What is the smallest safe test that would change my mind?
```

A good causal probe has five properties:

```text
1. It tests a specific relation.
2. It states a prediction in advance.
3. It avoids changing too many variables at once.
4. It defines what would count against the model.
5. It leads to an update.
```

Example:

```text
Weak AI use:
"Write me a better sales page."

H-AGI use:
"What is the main causal hypothesis behind this sales page?
What behaviour should improve if the hypothesis is right?
What one change can I test?
What metric would count as support?
What metric would weaken the hypothesis?
What will I do after the result?"
```

This keeps AI-assisted cognition coupled to the world.

# 16. Creativity protection track

AI can produce fluent, polished and average output. This creates a convergence trap. The user may feel creative because the output is smooth, while the underlying idea is generic.

H-AGI protects creativity through a structured divergence–constraint–probe cycle:

```text
human-first ideation
→ AI variation
→ cross-domain recombination
→ novelty check
→ usefulness check
→ boundary test
→ implementation probe
→ delayed originality re-check
```

The creativity rule is:

```text
Human first.
AI second.
Diversity before polish.
Constraint before action.
Probe before belief.
Delay before consolidation.
```

Before using AI for creative work:

```text
1. Generate 3–5 initial ideas or frames without AI.
2. Write them down.
3. Only then use AI for perturbation, recombination or cross-domain analogies.
4. Compare the AI output to the initial ideas.
5. Ask whether the AI has generated a genuinely new relation or merely polished an obvious one.
6. Select one idea for a boundary, audience or implementation probe.
```

## 16.1 Divergence-stuck protocol

If more than 7 options are active and the user cannot select:

```text
1. Force elimination: remove any option that does not change the predicted consequence.
2. Force comparison: pair options and ask "Which generates a more discriminating test?"
3. Force action: select the option with the smallest safe probe, regardless of perceived quality.
4. If still stuck: the problem is not creativity but decision-making. Route to Decide mode.
```

This prevents the creativity track from becoming a procrastination loop.

Mechanistically:

| Creativity step              | H-AGI mechanism                                        |
| ---------------------------- | ------------------------------------------------------ |
| Human-first ideation         | Local entropy generation from the user's own knowledge |
| AI variation                 | External entropy injection                             |
| Cross-domain recombination   | Relational alignment and analogical mapping            |
| Novelty check                | MI filter for non-redundancy                           |
| Usefulness check             | MI filter for relevance                                |
| Boundary test                | Map-boundary checking                                  |
| Implementation probe         | Causal testing                                         |
| Delayed originality re-check | Gc consolidation test                                  |

This protects against two opposite failures:

| Failure              | Description                      | Protocol response                                     |
| -------------------- | -------------------------------- | ----------------------------------------------------- |
| AI convergence trap  | Polished but generic output      | Human-first generation, novelty checks, varied frames |
| Unbounded divergence | Many ideas with no relevance     | MI filtering, constraints, implementation probes      |
| Untested novelty     | Interesting but unvalidated idea | Audience, behavioural or implementation test          |

Creativity is trained as:

```text
generate
→ vary
→ recombine
→ filter
→ select
→ test
→ update
→ consolidate
```

# 17. The H-AGI session structure

A standard H-AGI session can run in 15–30 minutes.

## 17.1 Progressive H-AGI tracks

| Track | Steps | For whom |
|-------|-------|----------|
| **Beginner** | 5-step simplified | New to structured AI use; build habit before complexity |
| **Standard** | 12-step full cycle | Trained users; full mechanism-aware session |
| **Advanced** | 12-step + wrapper variation + diagnostic scoring | Experienced users; targeting specific bottlenecks |

### Beginner track (5 steps)

```text
1. Human-first frame
   State the problem before using AI.

2. AI expansion
   Use AI to generate alternatives or structure.

3. Challenge
   Ask "What would make this wrong?"

4. One test
   Identify one small thing you can check or try.

5. Record
   Write down what you learned.
```

### Standard track (12 steps)

```text
0. Ψ-band / body-budget gate
   Check readiness using the operational gate (Section 4).

1. Human-first frame
   State the problem, first answer, assumptions and uncertainty before using AI.

2. Select thinking mode
   Choose Comprehend, Challenge, Probe, Decide, Plan, Reframe, Create or Learn.

3. Externalise the relational workspace
   Use AI or an artefact to map variables, relations, options and consequences.

4. Extract relations
   Identify the specific relations AI is proposing.
   If none are extractable, route to Comprehend (Section 11.1).

5. Near-miss foil
   Introduce plausible errors, hidden assumptions, wrong frames or causal traps.

6. Alignment and MI filter
   Run the 7-step MI filter (Section 11.2).
   Classify each relation: retain / revise / test / delay / reject.

7. SR-map update
   Use the template (Section 12).
   If no change occurred, use the null update protocol (Section 12.1).

8. Causal probe
   Identify the smallest safe test of the updated map.

9. Human judgement pass
   Accept, reject, revise, delay or test.

10. Niche action
    Deploy the output in a real task, decision, conversation, article, product or experiment.

11. Feedback update
    Record what happened, what was learned and what changed.

12. Delayed re-check
    Schedule a later re-check under changed conditions (Section 18).
```

Short version:

```text
frame
→ expand
→ extract
→ challenge
→ filter
→ update
→ probe
→ act
→ observe
→ re-check
```

# 18. Delayed re-checks and independence

Delayed re-checks are essential because immediate AI-supported performance may reflect scaffold dependence rather than learning.

A delayed re-check asks:

```text
Can I recover the same cognitive operation later?
Can I use it with a different AI model?
Can I use it in a different format?
Can I use it with no AI?
Can I apply it to a real-world case?
Can I explain the invariant relation?
```

There are three main delayed re-check types.

| Re-check type     | What it tests                               | Example                                                                                                                           |
| ----------------- | ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| Method recovery   | Can I perform the same reasoning operation? | Yesterday I used AI to detect hidden assumptions. Today, can I do it unaided on a new document?                                   |
| Map recovery      | Can I reconstruct the SR map?               | Yesterday I mapped a problem as A → B → C. Today, can I redraw that map and explain why B → C matters?                            |
| Transfer recovery | Can I apply the method to a new domain?     | Yesterday I used assumption detection on a business plan. Today, can I apply it to a research paper, lesson plan or product idea? |

Examples:

```text
Immediate task:
Use AI to identify assumptions in a business plan.

Delayed re-check:
Tomorrow, inspect a different plan without AI and identify the hidden assumptions.

Immediate task:
Use AI to reframe an article idea.

Delayed re-check:
Later, generate three new reframes without the same scaffold.

Immediate task:
Use AI to design a causal probe.

Delayed re-check:
In a new context, state the hypothesis, prediction, intervention and update rule unaided.
```

Only methods that survive delay, wrapper change and reduced scaffolding should be treated as consolidated.

## 18.1 Re-check failure protocol

If the user cannot recover the method, map or transfer:

```text
Method recovery fails:
  → Do not simply repeat the AI-assisted session.
  → Use Learn mode with AI to diagnose: "What did I miss? What was the key relation I failed to extract?"
  → Then: no-AI re-attempt with simplified material.
  → If still failing: the target may be too complex — reduce to a simpler relational target.

Map recovery fails:
  → The SR map was not truly updated in Session 1.
  → Return to SR-map update template and require more explicit revision.

Transfer recovery fails:
  → The relation was wrapper-dependent or domain-bound.
  → Apply horizontal variation with more diverse wrappers before re-check.
```

This prevents the user from either giving up or mindlessly repeating the same AI-assisted session.

# 19. Gf → Gc → Gf spiral

H-AGI follows the Trident-G developmental spiral.

## 19.1 Gf phase: AI-assisted expansion

In the fluid phase, the user searches a problem space. AI expands the range of possible relations, frames, analogies, plans, explanations and actions.

The risk is scatter, over-trust or fluency capture. The user therefore needs meta-epistemic control.

```text
AI expands the relation space.
The user extracts candidate relations.
Near-miss foils sharpen boundaries.
The MI filter selects what is relevant.
The SR map updates.
```

## 19.2 Causal probe: world feedback

The updated SR map is then tested against reality.

```text
What does the map predict?
What small action tests it?
What happened?
What changed?
```

## 19.3 Gc consolidation: reusable relational scripts

What survives becomes crystallised intelligence. But this Gc is not merely a prompt or workflow. It is a stabilised predictive-relational structure.

Examples:

```text
how to frame a messy problem
how to detect hidden assumptions
how to design a causal probe
how to map options to consequences
how to reframe a stuck problem
how to test a creative idea
how to update after feedback
```

The protocol should explicitly avoid confusing saved prompts with Gc.

| Not Gc                                  | Is Gc                                                                                                         |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| A prompt that gets AI to summarise well | The relational structure of a good summary: source → key claim → evidence → implication → audience need       |
| A workflow for writing articles         | The state-transition map of article production: problem → angle → evidence → structure → revision → feedback  |
| A list of questions to ask AI           | The invariant reasoning pattern: extract relations → test coherence → check boundaries → predict consequences |
| A favourite AI tool                     | A transferable method that survives tool change                                                               |
| A polished output                       | A reusable cognitive structure that improves later thinking                                                   |

Gc is **procedural knowledge about relations**, not merely procedural knowledge about tool use.

## 19.4 Gc maintenance

Consolidated scripts decay without use.

```text
Active scripts: re-check every 2–4 weeks (maintenance re-check)
Dormant scripts: re-check before first use after >1 month gap
Failed re-check: treat as new Gf target, not Gc recovery
```

This prevents the illusion that Gc, once formed, is permanent.

## 19.5 Later Gf: recombination

Later, the user re-enters the problem space with richer Gc. AI now augments a more intelligent controller, not a blank slate.

The spiral is:

```text
Gf searches.
AI expands.
Reasoning filters.
World feedback tests.
Gc stabilises.
Later Gf recombines.
```

In one sentence:

```text
H-AGI trains fluid intelligence by using AI to expand the relational search space, then converts what survives reasoning, causal probing and delayed transfer into portable crystallised intelligence.
```

# 20. Scoring and feedback

H-AGI should avoid claiming to directly measure or increase IQ. It can track process indicators that are theoretically meaningful and practically useful.

To keep the system usable, scoring should be tiered.

## 20.1 Tier 1: session routing

These measures guide what happens next in the current session.

| Measure               | What it tracks                                         |
| --------------------- | ------------------------------------------------------ |
| Human-first ratio     | How much thinking occurred before AI output            |
| Relation extraction   | Ability to identify the relations AI is proposing      |
| MI constraint quality | Ability to filter relevance from noise                 |
| Challenge quality     | Strength of counterexamples and disconfirming tests    |
| Ψ-band maintenance    | Whether the session remained sustainable and regulated |

## 20.2 Tier 2: progress tracking

These measures show whether the user is improving over time.

| Measure                 | What it tracks                                                 |
| ----------------------- | -------------------------------------------------------------- |
| Framing quality         | Clarity of problem, variables and constraints                  |
| SR-map update quality   | Whether the predictive map actually changed                    |
| Experimental discipline | Ability to form predictions, choose probes and avoid confounds |
| Update quality          | Whether feedback changes beliefs or actions                    |
| Transfer recovery       | Reuse across tools, wrappers and contexts                      |
| Delayed independence    | Recovery of the method without the same scaffold               |

## 20.3 Tier 3: diagnostic measures

These are used when a bottleneck is identified.

| Measure                   | What it tracks                                              |
| ------------------------- | ----------------------------------------------------------- |
| Entropy exploration depth | Breadth of useful search before convergence                 |
| Creativity diversity      | Range and non-redundancy of ideas                           |
| Convergence quality       | Selection of useful, feasible ideas                         |
| Evidence discipline       | Source-checking, uncertainty handling and claim calibration |
| Assumption detection      | Ability to find hidden premises                             |

## 20.4 Composite profile format

The H-AGI profile combines measures into an actionable summary.

Example:

```text
H-AGI composite profile

Framing:          ████████░░  8/10  (strong human-first, clear variables)
Reasoning:        ██████░░░░  6/10  (good extraction, weak alignment)
Challenge:        ████░░░░░░  4/10  (accepts fluency too easily — target for foils)
Evidence:         ███████░░░  7/10  (good source discipline)
Experiment:       █████░░░░░  5/10  (probes designed but rarely executed)
Decision:         ████████░░  8/10  (strong comparative evaluation)
Creativity:       ██████░░░░  6/10  (diverse but weak convergence)
Transfer:         ████░░░░░░  4/10  (wrapper-dependent — target for variation)
Independence:     ███░░░░░░░  3/10  (high scaffold dependence — target for delayed re-checks)
State regulation: ███████░░░  7/10  (good Ψ-band maintenance)

Bottleneck: Challenge + Transfer + Independence
Next session route: Full cycle with increased foils, wrapper variation, and delayed re-checks.
```

The purpose of scoring is not to label the person. It is to route the next session and identify bottlenecks.

# 21. Bottlenecks and adaptive routing

H-AGI can identify bottlenecks across the full cognitive-AI loop.

Possible bottleneck layers:

| Bottleneck              | Signs                                            | Route                                               |
| ----------------------- | ------------------------------------------------ | --------------------------------------------------- |
| Readiness bottleneck    | Fatigue, strain, low vigilance, high impulsivity | Recover, activate or simplify                       |
| Framing bottleneck      | User asks AI before defining the problem         | Human-first framing scaffold                        |
| Relation bottleneck     | User cannot extract what AI is proposing         | Relation extraction drills                          |
| Challenge bottleneck    | User accepts fluent output too easily            | Near-miss foils and red-team prompts                |
| Evidence bottleneck     | Weak source discipline or over-trust             | Source-checking and confidence calibration          |
| Causal bottleneck       | No testable prediction or probe                  | Causal board and smallest-test routine              |
| Creativity bottleneck   | Generic AI output or unbounded novelty           | Human-first ideation plus novelty/usefulness checks |
| Decision bottleneck     | Options generated but not compared               | Decision matrix or expected-value scaffold          |
| Transfer bottleneck     | Skill works only in one wrapper                  | Horizontal wrapper variation                        |
| Independence bottleneck | User cannot recover method without AI            | Delayed no-AI re-check                              |

## 21.1 Routing decision tree

```text
Session starts
→ Ψ-band gate

If readiness fails:
→ recover / activate / simplify

If readiness passes:
→ proceed to task

During task:

Can the user frame the problem?
→ No: use human-first framing scaffold
→ Yes: continue

Can the user extract relations?
→ No: use relation extraction drill
→ Yes: continue

Does the user accept AI output too easily?
→ Yes: add near-miss foils
→ No: continue

Is there no testable prediction?
→ Yes: use causal probe scaffold
→ No: continue

Is the output generic or over-convergent?
→ Yes: use creativity track
→ No: continue

Can the user decide between options?
→ No: use decision matrix
→ Yes: continue

Does the skill work only in one wrapper?
→ Yes: apply horizontal variation
→ No: continue

Can the user recover the method without AI?
→ No: use delayed re-check protocol
→ Yes: consolidate into Gc
```

The adaptive rule is:

```text
Proceed normally if the chain is intact.
Target the bottleneck if one layer repeatedly fails.
Reduce load if state readiness is poor.
Increase wrapper variation if surface fluency is too high.
Increase causal probing if output is plausible but untested.
Increase delayed re-checks if scaffold dependence is high.
```

# 22. Troubleshooting common H-AGI failures

## 22.1 AI gives contradictory advice

Do not average the outputs. Extract the relations.

Ask:

```text
What assumptions differ between the two answers?
What evidence would distinguish them?
Are they solving the same problem?
Which recommendation has a clearer causal path?
What small test would discriminate between them?
```

If contradictions come from different models:

```text
1. First check: is this a disagreement about facts or about values/framings?
2. If factual: which claim has stronger independent evidence?
3. If framing: which model's bias aligns with the task's requirements?
4. If uncertain: treat as a Challenge foil — the contradiction itself reveals an assumption gap.
5. Do not resolve by "majority vote" across models.
```

Route:

```text
Comprehend → Challenge → Probe
```

## 22.2 The user is stuck in a reframing loop

Reframing can become unbounded entropy if the user keeps generating new frames without selecting or testing.

Ask:

```text
Which frame changes the predicted consequence?
Which frame suggests a different action?
Which frame can be tested now?
```

Route:

```text
Reframe → MI filter → causal probe
```

## 22.3 AI output is fluent but generic

This is the AI convergence trap.

Ask:

```text
What is the cliché here?
What relation is genuinely new?
What did I generate before AI?
What constraint would force a less obvious answer?
What audience or implementation probe can test this?
```

Route:

```text
Create → novelty check → usefulness check → probe
```

## 22.4 The user cannot identify a causal probe

Some problems are too large or abstract. Reduce the scale.

Ask:

```text
What is the smallest claim inside this model?
What behaviour, signal or feedback would change if it were true?
Can I test a proxy without confusing it for the final outcome?
What would count against the claim?
```

Route:

```text
Probe scaffold → smallest safe intervention
```

## 22.5 The user keeps asking AI before thinking

This is passive offloading.

Ask:

```text
What is my first answer?
What do I already know?
What are my assumptions?
What relation am I asking AI to help with?
```

Route:

```text
Human-first frame → AI expansion only after initial attempt
```

## 22.6 The user becomes overloaded by AI output

Reduce entropy load.

Ask AI for:

```text
one frame
three variables
one assumption
one risk
one next test
```

Route:

```text
Simplify → scaffold → one-mode session
```

## 22.7 The user is fast, confident and brittle

Slow down commitment.

Ask:

```text
What would make this wrong?
What is the strongest counterexample?
What hidden assumption am I skipping?
What should I delay until after a re-check?
```

Route:

```text
Slow-check → Challenge → delayed action
```

## 22.8 The user becomes emotionally attached to AI output

The user defends an AI-generated idea because they invested time in it, not because it is correct.

Ask:

```text
Would I defend this idea if a colleague proposed it?
What would make me abandon this idea?
What evidence, if it appeared tomorrow, would change my mind?
Has the AI become a teammate I am protecting, or a tool I am testing?
```

Route:

```text
Challenge → red-team foil → Probe
```

## 22.9 The user uses AI to avoid difficult decisions

The user keeps generating more options, analyses or reframes rather than committing.

Ask:

```text
What is the cost of deciding now versus delaying?
What is the smallest reversible commitment I can make?
What would I decide if I had no AI access?
Is this analysis or procrastination?
```

Route:

```text
Decide → forced choice → smallest safe commitment
```

# 23. H-AGI as anti-offloading training

The central risk of AI is not simply that it gives wrong answers. The deeper risk is that it can allow users to bypass the cognitive operations that build intelligence.

H-AGI prevents this through five anti-offloading rules:

```text
1. Human-first framing:
   The user states the problem before AI expands it.

2. Relation extraction:
   The user identifies what AI is actually claiming.

3. Challenge before acceptance:
   Plausible output is tested with foils.

4. Probe before belief:
   Important claims are linked to predictions or feedback loops.

5. Delayed independence:
   The method is re-tested without the same scaffold.
```

This turns AI use into cognitive training.

The user does not merely consume outputs. The user practises:

```text
framing
reasoning
assumption checking
counterfactual thinking
causal testing
decision-making
creative recombination
feedback updating
transfer
```

# 25. Final principle

H-AGI trains users to use AI as a dynamic cognitive niche while preserving human-led control over intelligence.

AI is not treated as an oracle. It is treated as an entropy source, simulation engine, critique partner, artefact generator and external workspace.

The human is not treated as a passive consumer. The human is the controller who frames the problem, extracts relations, filters relevance, tests assumptions, chooses probes, interprets feedback and consolidates what survives.

The world is not treated as an afterthought. It is the testing ground that disciplines AI-assisted cognition.

Final compact statement:

```text
AI injects entropy into the relational workspace.
Human reasoning aligns and filters candidate relations.
The SR map is updated only when the new relation improves prediction.
The world tests the map through causal probes.
What survives is consolidated into portable Gc.
Later Gf re-enters with a richer predictive map.
```

In one sentence:

**H-AGI trains users to maintain a near-critical, body-budget-supported regime in which AI-assisted exploration is selectively coordinated into adaptive SR-map updates, disciplined by near-miss foils, causal probes, wrapper variation and delayed re-checks, producing transferable cognitive restructuring rather than tool-dependent output generation.**
