# High-Level Mindware Meta-Functions for H-AGI Tools

## Core idea

A **mindware meta-function** is a reusable cognitive control system for a broad class of human problems.

Each meta-function uses the same underlying G-loop:

```text
+ F★ protection
+ λ support estimation
+ 𝒢-volume direction
+ MI target construction
+ Gf model-building / inference
+ SR horizon selection
+ scale-appropriate feasible closure
+ niche perturbation
+ prediction-error evaluation
+ deep Gc banking
= strategic spiral advance
```

But each meta-function has a different **object of resolution**.

| Meta-function                  | Object of resolution                   |
| ------------------------------ | -------------------------------------- |
| **Comprehension**              | A usable situation model               |
| **Argumentation**              | A defensible claim structure           |
| **Persuasion / communication** | A message that can shift another model |
| **Decision-making**            | A selected action under uncertainty    |
| **Strategic action**           | A position-improving path              |
| **Negotiation**                | A workable agreement or next probe     |
| **Learning / transfer**        | A portable invariant                   |
| **Reappraisal / resilience**   | An updated appraisal model             |

In H-AGI terms, the AI tool should not simply “give answers”. It should help the human run the correct meta-function more reliably, with better working-memory support, better alternatives, better tests, and better Gc banking.

# 1. Comprehension / Sensemaking

## Core function

Comprehension is the meta-function for turning information into a usable model.

It asks:

```text
What is going on here?
What are the main variables?
How do the parts relate?
What matters, what is background, and what follows?
```

## G-loop interpretation

| G-loop component                | Comprehension version                                                                                  |
| ------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **F★ protection**               | Avoid overload, confusion, premature certainty or passive acceptance.                                  |
| **λ support estimation**        | Check whether the source, context and evidence are adequate.                                           |
| **𝒢-volume direction**         | Build a model that can be used beyond the immediate text or situation.                                 |
| **MI target construction**      | Define what “understanding” means: summary, causal model, mechanism, implications, or critique.        |
| **Gf model-building**           | Identify variables, relations, mechanisms, assumptions, gaps and implications.                         |
| **SR horizon selection**        | Decide whether the model needs immediate meaning, medium-term implications or long-range consequences. |
| **Closure point**               | Produce a clear summary, model, diagram, teaching explanation, or set of questions.                    |
| **Niche perturbation**          | Use the model: explain it, apply it, teach it, ask a better question.                                  |
| **Prediction-error evaluation** | Does the model help prediction, explanation or action?                                                 |
| **Deep Gc banking**             | Bank the schema: “This is a case of X pattern.”                                                        |

## AI-enhanced tool

A **Comprehension GPT** should:

```text
extract the core claim or mechanism
identify variables and relations
separate main point from detail
create layered summaries
map causes, consequences and assumptions
ask what would follow if the model were true
generate teaching explanations
produce diagrams or structured notes
bank the generalisable schema
```

## Deep Gc output

```text
A portable model, explanatory schema, concept map, or reusable comprehension prompt.
```

Example banked prompt:

```text
What is the relation being proposed, and what would change if that relation were false?
```

# 2. Argumentation / Critical Evaluation

## Core function

Argumentation is the meta-function for constructing, evaluating and improving claims.

It asks:

```text
What is being claimed?
What reasons support it?
What assumptions does it depend on?
What would weaken or defeat it?
```

## G-loop interpretation

| G-loop component                | Argumentation version                                                                        |
| ------------------------------- | -------------------------------------------------------------------------------------------- |
| **F★ protection**               | Avoid ego-defence, overconfidence, motivated reasoning or adversarial collapse.              |
| **λ support estimation**        | Check evidence quality, source reliability, context and standards of proof.                  |
| **𝒢-volume direction**         | Improve belief quality, reasoning power and public credibility.                              |
| **MI target construction**      | Define what success means: valid argument, strong evidence, persuasive case, fair critique.  |
| **Gf model-building**           | Map premises, conclusions, assumptions, counterarguments, alternatives and inferential gaps. |
| **SR horizon selection**        | Consider immediate rebuttal, medium-term debate, or long-term belief revision.               |
| **Closure point**               | Produce an argument tree, critique, revised thesis or stronger version.                      |
| **Niche perturbation**          | Publish, teach, debate, submit, test with audience or use in discussion.                     |
| **Prediction-error evaluation** | Which objections landed? Which evidence changed the claim?                                   |
| **Deep Gc banking**             | Bank better argument patterns and failure signatures.                                        |

## AI-enhanced tool

An **Argumentation GPT** should:

```text
identify main conclusion
separate premises from assumptions
map counterarguments
distinguish logical weakness from evidence weakness
steelman and stress-test claims
show where confidence should be qualified
generate argument trees
suggest better evidence
produce revised claims with boundaries
```

## Deep Gc output

```text
A reusable argument map, reasoning principle, boundary condition, or critical-thinking prompt.
```

Example banked prompt:

```text
What must be true for this conclusion to follow?
```

# 3. Persuasion / Communication

## Core function

Persuasion is the meta-function for shifting another person’s model, attention or action without merely transmitting information.

It asks:

```text
Whose model needs to change?
What do they currently believe, value, fear or misunderstand?
What message could move them one step?
```

## G-loop interpretation

| G-loop component                | Persuasion version                                                                      |
| ------------------------------- | --------------------------------------------------------------------------------------- |
| **F★ protection**               | Avoid reputational damage, manipulation, overclaiming or audience alienation.           |
| **λ support estimation**        | Estimate audience, channel, timing, relationship and trust conditions.                  |
| **𝒢-volume direction**         | Build authority, alignment, trust, audience growth or action-readiness.                 |
| **MI target construction**      | Define the desired shift: belief, attention, trust, decision, behaviour or curiosity.   |
| **Gf model-building**           | Model the audience’s current frame, objections, incentives and emotional state.         |
| **SR horizon selection**        | Decide whether this is a one-message, campaign, relationship or brand-positioning loop. |
| **Closure point**               | Produce a post, email, pitch, explainer, offer, CTA or narrative frame.                 |
| **Niche perturbation**          | Send, publish, present, test, ask for response.                                         |
| **Prediction-error evaluation** | Did the audience respond, ignore, resist, trust, ask, click or act?                     |
| **Deep Gc banking**             | Bank message patterns, audience maps and positioning rules.                             |

## AI-enhanced tool

A **Persuasion GPT** should:

```text
model audience beliefs and objections
translate technical ideas into user-facing language
generate alternative frames
check for overclaiming
identify emotional and practical barriers
produce message variants
test clarity and resistance points
generate CTAs
bank what worked by audience type
```

## Deep Gc output

```text
A reusable audience model, message frame, objection map, or positioning principle.
```

Example banked prompt:

```text
What model does the audience currently have, and what is the smallest shift this message should produce?
```

# 4. Decision-Making

## Core function

Decision-making is the meta-function for selecting action under uncertainty.

It asks:

```text
What choice is actually being made?
What options exist?
What matters most?
What is likely to happen if I choose each path?
```

## G-loop interpretation

| G-loop component                | Decision-making version                                                                                    |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **F★ protection**               | Avoid catastrophic downside, exhaustion, sunk-cost escalation or avoidable fragility.                      |
| **λ support estimation**        | Check resources, timing, authority, information quality and reversibility.                                 |
| **𝒢-volume direction**         | Choose in a way that expands future options rather than merely resolving discomfort.                       |
| **MI target construction**      | Define the decision criterion: expected value, risk reduction, learning value, optionality, ethics, speed. |
| **Gf model-building**           | Generate options, trade-offs, causal consequences, unknowns and counterfactuals.                           |
| **SR horizon selection**        | Decide whether to optimise for immediate, short-term, project-level or long-run consequences.              |
| **Closure point**               | Decide, defer, gather evidence, run a test, delegate or kill the option.                                   |
| **Niche perturbation**          | Make the decision visible: commit, test, communicate, purchase, reject, schedule.                          |
| **Prediction-error evaluation** | Did the choice work as expected? What was misestimated?                                                    |
| **Deep Gc banking**             | Bank decision rules, heuristics, base rates and known failure modes.                                       |

## AI-enhanced tool

A **Decision GPT** should:

```text
clarify the decision
separate options from criteria
estimate reversibility
surface hidden assumptions
run pre-mortems
identify information value
compare trade-offs
produce decision memos
track outcomes against expectations
bank improved decision rules
```

## Deep Gc output

```text
A decision rule, risk model, base-rate estimate, pre-mortem pattern or choice heuristic.
```

Example banked prompt:

```text
Is this a reversible test, an irreversible commitment, or a delay-cost problem?
```

# 5. Strategic Action

## Core function

Strategic action is the meta-function for improving future position.

It asks:

```text
What result will improve security, options, leverage, capability or niche position?
What is the right-sized closure point?
What should be banked?
```

## G-loop interpretation

| G-loop component                | Strategic action version                                                                |
| ------------------------------- | --------------------------------------------------------------------------------------- |
| **F★ protection**               | Protect the base: health, money, attention, reputation, core systems.                   |
| **λ support estimation**        | Check whether current resources and niche conditions support the chosen scale.          |
| **𝒢-volume direction**         | Define the larger arc: options, capability, leverage, security.                         |
| **MI target construction**      | Specify the result that would actually improve position.                                |
| **Gf model-building**           | Model constraints, routes, dependencies, bottlenecks, leverage points and alternatives. |
| **SR horizon selection**        | Choose micro, session, daily, weekly, project, developmental or life-arc scale.         |
| **Closure point**               | Select the next appropriate completion point at that scale.                             |
| **Niche perturbation**          | Publish, deploy, sell, contact, decide, test, repeat.                                   |
| **Prediction-error evaluation** | Did position improve or merely activity increase?                                       |
| **Deep Gc banking**             | Bank assets, routines, strategic rules, maps and objective priors.                      |

## AI-enhanced tool

A **Strategic Action GPT** should:

```text
identify the strategic arc
turn ambition into a valid objective
detect overlarge or underpowered plans
choose right-sized closure points
run pre-mortems
track position change
force external action
extract bankable structures
reduce idea-fragmentation
maintain a memory of strategic priors
```

## Deep Gc output

```text
A strategic asset, operating rule, opportunity map, partnership map, execution routine or constraint.
```

Example banked prompt:

```text
What is the next appropriate closure point at this scale?
```

# 6. Negotiation

## Core function

Negotiation is the meta-function for reaching agreement, extracting information, or improving relational position under partly conflicting interests.

It asks:

```text
What do they need?
What do I need?
What can be traded, tested, clarified or refused?
What agreement or next probe improves position?
```

## G-loop interpretation

| G-loop component                | Negotiation version                                                                                       |
| ------------------------------- | --------------------------------------------------------------------------------------------------------- |
| **F★ protection**               | Protect minimum acceptable terms, reputation, relationship, time, legal and emotional boundaries.         |
| **λ support estimation**        | Estimate trust, power, alternatives, dependency, timing and communication channel.                        |
| **𝒢-volume direction**         | Improve relational options, deal quality, leverage, trust or future cooperation.                          |
| **MI target construction**      | Define success: agreement, information, no-deal, better BATNA, next meeting, clarified interests.         |
| **Gf model-building**           | Model interests, constraints, alternatives, incentives, fears, tradeable variables and hidden blockers.   |
| **SR horizon selection**        | Decide whether this is a single exchange, multi-step negotiation, relationship arc or strategic alliance. |
| **Closure point**               | Ask a probe, make an offer, request terms, set a boundary, propose a trade.                               |
| **Niche perturbation**          | Send offer, have call, ask question, test flexibility, pause or walk away.                                |
| **Prediction-error evaluation** | What did their response reveal about interests, limits or trust?                                          |
| **Deep Gc banking**             | Bank negotiation patterns, offer structures, boundary rules and relationship maps.                        |

## AI-enhanced tool

A **Negotiation GPT** should:

```text
map interests and BATNAs
identify tradeable variables
generate probe questions
draft offers and counteroffers
simulate the other party’s likely model
flag weak positions and overconcessions
prepare fallback options
extract lessons from responses
bank negotiation scripts and boundary conditions
```

## Deep Gc output

```text
A negotiation map, offer template, boundary rule, relationship model or leverage principle.
```

Example banked prompt:

```text
Is this a price problem, trust problem, authority problem, timing problem or value-discrimination problem?
```

# 7. Learning / Transfer

## Core function

Learning / transfer is the meta-function for extracting what generalises.

It asks:

```text
What can I learn from this that travels?
What stayed invariant when the surface changed?
Where else does this apply?
```

## G-loop interpretation

| G-loop component                | Learning / transfer version                                                            |
| ------------------------------- | -------------------------------------------------------------------------------------- |
| **F★ protection**               | Avoid overload, shallow repetition, demoralisation or brittle overtraining.            |
| **λ support estimation**        | Check practice quality, feedback quality, recovery and real-world relevance.           |
| **𝒢-volume direction**         | Build capability that expands future problem-solving territory.                        |
| **MI target construction**      | Define what should transfer: relation, rule, method, schema, routine, prompt.          |
| **Gf model-building**           | Identify variables, relations, examples, counterexamples, analogies and boundaries.    |
| **SR horizon selection**        | Decide whether the learning is immediate, weekly, project-level or developmental.      |
| **Closure point**               | Complete practice block, wrapper swap, test, explanation, mission or delayed re-check. |
| **Niche perturbation**          | Use the skill in a changed task, real setting, teaching context or applied mission.    |
| **Prediction-error evaluation** | Did the structure survive variation, delay and real use?                               |
| **Deep Gc banking**             | Bank the invariant, boundary, prompt or routine that survived.                         |

## AI-enhanced tool

A **Transfer Coach GPT** should:

```text
identify the deep relation
generate varied examples
create near-miss cases
swap wrappers
test boundary conditions
ask for self-explanation
schedule delayed re-checks
link the skill to real-world missions
help bank the invariant
```

## Deep Gc output

```text
A portable invariant, transfer principle, boundary condition, or reusable learning routine.
```

Example banked prompt:

```text
What changed on the surface, and what relation stayed the same?
```

# 8. Reappraisal / Cognitive Resilience

## Core function

Reappraisal is the meta-function for updating emotionally loaded predictive models.

It asks:

```text
What prediction am I making?
What else could this mean?
What small test could update the model?
```

## G-loop interpretation

| G-loop component                | Reappraisal version                                                                      |
| ------------------------------- | ---------------------------------------------------------------------------------------- |
| **F★ protection**               | Avoid destabilising exposure, rumination, over-control or unsafe action.                 |
| **λ support estimation**        | Check body state, emotional load, support, context and safety.                           |
| **𝒢-volume direction**         | Expand flexibility under stress, criticism, pain, uncertainty or performance pressure.   |
| **MI target construction**      | Define the target: reduce fusion, test prediction, choose adaptive action, update model. |
| **Gf model-building**           | Separate trigger, body signal, emotion, meaning, prediction, evidence and action urge.   |
| **SR horizon selection**        | Choose immediate regulation, small test, repeated practice or longer schema update.      |
| **Closure point**               | Select a small probe, reframing statement, behavioural test, implementation intention.   |
| **Niche perturbation**          | Act differently, test safely, log outcome, practise routine.                             |
| **Prediction-error evaluation** | Did the outcome match the threat model or update it?                                     |
| **Deep Gc banking**             | Bank a new appraisal map or recovery routine.                                            |

## AI-enhanced tool

A **Resilience / Reappraisal GPT** should:

```text
separate sensation, emotion, prediction and meaning
identify hidden appraisal models
generate alternative interpretations
select small safe probes
create implementation intentions
track outcomes
detect threat-lure errors
bank updated predictive maps
avoid clinical overclaiming
```

## Deep Gc output

```text
A safer predictive map, recovery routine, reappraisal prompt or implementation cue.
```

Example banked prompt:

```text
What prediction is this feeling making, and what small safe test could update it?
```

# Cross-function architecture

All H-AGI mindware tools should share the same underlying architecture:

```text
1. Readiness / support check
2. F★ protection
3. Larger arc or purpose
4. MI target definition
5. Gf model-building
6. SR horizon selection
7. Right-sized closure
8. Niche action / output
9. Prediction-error evaluation
10. Deep Gc banking
```

But each tool specialises the loop around a different resolution object.

| Tool family              | Resolution object       | Typical output                          |
| ------------------------ | ----------------------- | --------------------------------------- |
| **Comprehension GPT**    | Situation model         | Summary, map, mechanism, questions      |
| **Argumentation GPT**    | Claim structure         | Argument tree, critique, revised thesis |
| **Persuasion GPT**       | Audience shift          | Message, frame, CTA, objection map      |
| **Decision GPT**         | Chosen action           | Decision memo, trade-off table, test    |
| **Strategic Action GPT** | Position gain           | Closure point, action plan, asset bank  |
| **Negotiation GPT**      | Agreement / probe       | Offer, BATNA map, trade structure       |
| **Transfer GPT**         | Portable invariant      | Wrapper tests, examples, banked rule    |
| **Resilience GPT**       | Updated appraisal model | Probe, reframe, implementation cue      |

# H-AGI design principle

The human and AI should not be treated as separate intelligences where the AI simply supplies outputs. The tool should create a coupled human–AI G-loop.

## Human role

```text
values
stakes
lived context
ethical judgement
body-state awareness
final decision
real-world action
ownership of banked Gc
```

## AI role

```text
external working memory
alternative models
counterarguments
scenario generation
pre-mortems
structured prompts
drafts and diagrams
boundary tests
feedback logging
banked-rule extraction
```

## Shared H-AGI loop

```text
human context
→ AI-supported modelling
→ human judgement
→ AI-supported testing
→ human action
→ real-world feedback
→ AI-supported reflection
→ human-owned deep Gc
```

The goal is not AI substitution. The goal is to strengthen the human’s ability to run better G-loops.

# Minimum viable GPT pattern for each meta-function

Every specialised GPT should include the same internal structure:

```text
1. Clarify the function
   Are we comprehending, arguing, deciding, negotiating, persuading, learning, reappraising or acting strategically?

2. Protect the base
   What must not be damaged?

3. Define the target
   What would count as success?

4. Build the model
   What variables, relations, causes, constraints and paths matter?

5. Choose the horizon
   What scale of loop are we running?

6. Select closure
   What is the next appropriate output or action?

7. Test
   What would make this wrong, weak, risky or non-transferable?

8. Produce
   Generate the map, argument, message, decision, plan, offer, probe or action.

9. Evaluate
   What feedback should be checked?

10. Bank
   What portable rule, prompt, asset, schema or map should carry forward?
```
