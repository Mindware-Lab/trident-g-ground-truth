# Trident-G Far Transfer Protocol

## A mechanistic outline for Zone, G, and R training protocols

### Core thesis

The deeper far-transfer hypothesis is:

> **Far transfer is not produced by repeating a cognitive task. It is produced by maintaining the learner in a trainable near-critical regime, tuning predictive relational maps across surface variation, forcing controlled breakpoint transitions before surface-specific automation hardens, validating portability across wrappers, strengthening cross-layer control policies, coupling those policies to real-world niche cues and feedback, and allowing consolidation so that cross-context invariants become slow schematic Gc.**

In Trident-G terms, this is training the slow Gc schematic pathway by repeatedly forcing re-entry into the Ψ-band after local SR automation, while also strengthening the vertical control chain that links attention control, working memory, SR-style inference, language-accessible strategy prompts, implementation intentions, real-world deployment, and niche-coupled feedback.

The framework now has **three applications**:

| Application                               | Protocol                            | Primary function                                                                                                                            |
| ----------------------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Zone / state–niche readiness training** | Zone Coach                          | trains readiness, attention-control capacity, Ψ-band access, recovery, and external niche gating                                            |
| **G / adaptive intelligence training**    | WM–Puzzle Epistemic Transfer        | trains variable abstraction, SR-style reasoning operators, problem-space navigation, epistemic prompts, wrapper transfer and niche transfer |
| **R / cognitive resilience training**     | WM–Reappraisal Cognitive Resilience | trains adaptive reappraisal, threat-schema updating, recovery under stress/load, and real-life implementation in emotionally salient niches |


So the compact formulation becomes:

```text
Zone trains the state and gates the niche.
G trains the structure.
R trains resilient reappraisal.
Prompts train the thinker.
Implementation intentions train deployment.
Reinforcement binds the verticals.
Wrapper swaps test the horizontals.
Niche probes test real-world coupling.
Delayed probes test slow Gc.
```

A fuller formulation is:

> **Horizontal transfer trains invariant recovery across wrappers. Vertical transfer trains the reinforced control chain linking state, attention, WM/SR inference, language-accessible epistemic control, implementation intentions, and real-world deployment.**

This aligns with the Trident-G definition of adaptive intelligence as maintaining near-critical cognition, searching relational problem spaces, validating structures across variation, and crystallising what survives into reusable Gc. It also fits the source theory’s distinction between α-stability monitoring, hippocampal SR maps, fast and slow Gc construction, and allostatic regulation through the Ψ-band.

A third transfer dimension is also needed:

Niche transfer trains the coupling between the learner’s internal control policy and the external environment in which that policy must operate.

A skill has not fully transferred merely because it survives a changed task wrapper or can be reconstructed through a vertical control chain. It must also remain coupled to the real-world niche: the relevant cues, affordances, constraints, feedback loops, social meanings, risks, rewards and action consequences that make the policy usable outside the training environment.

So the full transfer architecture is:

**horizontal transfer**
= invariant recovery across wrappers

**vertical transfer**
= control-policy reconstruction across cognitive layers

**niche transfer**
= real-world coupling across affordances, cues, feedback and consequences

# 1. The theoretical base

## 1.1 Trident-G: intelligence as Ψ-band navigation and spiral advance

Trident-G treats adaptive intelligence as a dynamical process, not a static capacity. The G-loop depends on:

```text
Ψ-band access
+ entropy–mutual information balance
+ niche coupling
+ SR map tuning
+ fast Gc compilation
+ slow Gc schematic extraction
+ horizontal portability validation
+ vertical control-policy integration
+ niche transfer validation
+ consolidation
```

The key distinction is between **flat task improvement** and **spiral advance**.

Flat improvement means the learner becomes better at the task surface. Spiral advance means the learner returns to later problems with a richer representational starting point because slow Gc has installed a deeper relational invariant.

The revised framework adds a second distinction:

```text
Horizontal transfer
= the same invariant survives across wrappers.

Vertical transfer
= the same adaptive control policy is reconstructed across cognitive layers.

Niche transfer
= the same policy remains coupled to real-world affordances, cues, feedback, constraints and consequences.
```

So transfer is not only about whether the same relation survives a new format. It is also about whether the learner can connect:

```text
state regulation
→ attention control
→ working-memory binding
→ SR-style inference
→ mission application
→ meta-epistemic prompt
→ implementation intention
→ real-world action
→ delayed re-use
```
But this vertical chain only becomes fully transferable when it is coupled to the niche:

```text
real-world cue
→ policy activation
→ action
→ environmental feedback
→ update
→ delayed re-use under changed niche conditions
```
This is why implementation intentions are important but not sufficient. They bind a cue to a planned response, but the cue must be ecologically valid. A good implementation intention fires in the real niche, not only in the learner’s abstract plan.

## 1.1.1 Niche coupling: the missing transfer condition

Trident-G treats adaptive intelligence as Ψ-band navigation, but the Ψ-band is not an isolated internal state. It is sustained in relation to a niche: the structured environment of affordances, cues, constraints, feedback loops, social meanings, risks and possible actions within which cognition must operate.

In Trident-G terms, λ indexes niche coupling: the quality of the link between internal GP dynamics and external environmental structure. High λ means the learner’s internal search space is well coupled to the niche’s real affordances and feedback. Low λ means the entropy arm may search internally, but the search does not accurately reflect what the world is actually asking, offering or constraining.

This matters for far transfer because a control policy can appear successful inside a training task while failing to couple to the real-world niche. The learner may know the prompt, solve the puzzle, or complete the reappraisal exercise, but fail to recognise the relevant cue in ordinary life, fail to act at the right moment, or fail to obtain useful feedback.

The framework therefore distinguishes:

**task success**
= performance improves inside the training surface

**wrapper transfer**
= the same invariant survives changed task form

**vertical transfer**
= the same policy can be reconstructed across state, attention, WM/SR inference, prompt and action

**niche transfer**
= the policy remains usable in a real-world environment with live cues, stakes, noise, feedback and consequences

## 1.2 Zhang–Tang: learning as entropy constrained by mutual information

Zhang and Tang’s model gives a formal analogue of the Trident-G explore–exploit dynamic. Their PNAS paper describes learning as a dynamic balance between maximum entropy, which supports broad exploration, and mutual-information constraint, which keeps updates relevant to the task objective. They report heavy-tailed update distributions and critical-like training dynamics in neural networks. ([PubMed][1])

In Trident-G language:

| Zhang–Tang construct             | Trident-G interpretation                       |
| -------------------------------- | ---------------------------------------------- |
| Maximum entropy                  | broad relational search / entropy arm          |
| Mutual information constraint    | relevance filtering / constraint arm           |
| Heavy-tailed update distribution | productive near-critical learning              |
| Large intermittent updates       | restructuring / bifurcation events             |
| Collapse into narrow constraint  | premature exploitation / brittle Gc            |
| Excess entropy                   | scattered exploration / failure to consolidate |

So the target is not maximum exploration and not rigid optimisation. The target is **productive tension between exploration and constraint**.

## 1.3 Successor representations: the right representational target

Successor representations are important because they encode states in terms of their likely future occupancy, not merely as isolated current stimuli. Stachenfeld, Botvinick, and Gershman describe the hippocampus as a predictive map and argue that hippocampal representations are predictive, reward-sensitive and policy-dependent rather than purely spatial. ([Nature][2])

That makes SRs ideal for far-transfer training because transfer depends on learning:

```text
state → transition → consequence → next reachable state
```

not merely:

```text
item → response
```

In the revised framework, SRs support both:

```text
horizontal transfer
= the predictive map survives wrapper changes

vertical transfer
= the predictive map becomes usable by higher-level strategy scripts and real-world action policies
```

## 1.4 Donner–Hardy: why breakpoints matter

Donner and Hardy’s piecewise power-law account supports the idea that individual learning is not always one smooth curve. The training-relevant implication is that local improvement phases may be punctuated by strategy shifts, reorganisations or breakpoint transitions.

This gives the protocol a behavioural signature:

```text
local tuning
→ asymptote / diminishing update
→ perturbation
→ dip
→ recovery
→ improved portability
```

The key move is to perturb the surface **near local asymptote**, before the learner fully banks a brittle surface strategy.

## 1.5 Why this is different from ordinary WM training

The framework explicitly avoids the claim that ordinary working-memory training reliably increases intelligence. Melby-Lervåg, Redick, and Hulme found reliable near transfer to similar working-memory tasks, but no convincing far transfer to broader abilities when compared with treated controls. ([Sage Journals][3])

So the claim is not:

> WM training improves g.

The claim is:

> WM provides the active workspace in which variables, relations, counterexamples, constraints, appraisals and alternative representations can be held and updated while SR-style inferential maps are trained across wrappers and connected to language-accessible control policies.

This is why WM training alone is not enough. In the Trident-G protocol, WM is embedded inside a broader transfer architecture:

```text
Zone gate
+ WM/SR operator training
+ puzzle or reappraisal mission
+ meta-epistemic prompt
+ wrapper swap
+ implementation intention
+ delayed re-check
```

## 1.6 Evidence for vertical-transfer components

The full Trident-G vertical-transfer architecture has not yet been directly validated as a complete protocol. However, its component mechanisms have empirical support.

Metacognitive and self-regulated strategy instruction improves academic performance across domains, with meta-analytic evidence showing substantial effects in writing, science, mathematics and reading. ([Groningen Research Portal][4]) Self-explanation prompts also show a positive overall effect, with one meta-analysis reporting a weighted mean effect of **g = .55** across 69 effect sizes. ([ResearchGate][5])

There is also evidence that strategy-based or metacognitively enriched working-memory training can outperform skill practice alone in some contexts. A double-blind RCT found that Cogmed plus metacognitive strategy training produced stronger maintained WM gains than Cogmed alone, although the authors still called for further work on far transfer. ([PubMed][6]) A smaller strategy-based WM-training study found transfer to a novel problem-solving task in children. ([Frontiers][7])

Inductive-reasoning training is another relevant precedent. Klauer and Phye’s review/meta-analysis summarised 74 training experiments and reported support for procedural comparison-based reasoning training improving fluid-intelligence test performance and academic learning. ([Sage Journals][8])

Implementation intentions provide especially strong support for the real-world deployment layer. Gollwitzer and Sheeran’s meta-analysis found that implementation intentions had a medium-to-large effect on goal attainment, **d = .65**, across 94 independent tests. ([ScienceDirect][9])

So the defensible empirical claim is:

> **There is strong converging evidence for the components of vertical transfer — strategy instruction, self-explanation, metacognitive prompting, strategy-enriched skill training and implementation intentions — but the full Trident-G CCC → WM/SR → mission → prompt → implementation → delayed consolidation stack remains an evidence-generating hypothesis.**

# 2. Core mechanism: breakpoint-driven relational SR training

The central horizontal training cycle is:

```text
1. Hold invariant relational structure constant.
2. Vary surface wrappers within a modality.
3. Detect local asymptote or diminishing update.
4. Perturb the surface or modality while preserving the invariant.
5. Look for initial dip followed by faster recovery.
6. Validate portability across wrappers and boundary cases.
7. Deploy the policy through a real-world or simulated niche cue.
8. Observe whether action produces useful environmental feedback.
9. Re-test after delay and under changed niche conditions.
10. Only then bank the structure as a slow schematic Gc candidate.
```
The key point is that far transfer is not validated by wrapper survival alone. A policy must also survive niche contact. It must be triggered by real cues, guide action under real constraints, and update through real feedback. Without this, the learner may possess a portable representation but not a deployable adaptive policy.

The mechanism can be stated as:

> **Standard training often produces fast Gc: efficient, task-specific compilation. Far-transfer training should repeatedly force the learner through controlled local bifurcations, where a surface policy fails but the deeper relational invariant remains recoverable. Each successful recovery tunes an SR over the invariant rather than over the wrapper. Across repetitions and consolidation intervals, the slow Gc pathway extracts a cross-domain schematic geometry.**

This is the distinction between **surface skill**, **fast Gc**, **slow schematic Gc**, and **Gf re-entry**.

| Learning product  | Mechanism                          | Transfer expectation     |
| ----------------- | ---------------------------------- | ------------------------ |
| Surface skill     | stimulus–response tuning           | low                      |
| Fast Gc           | efficient strategy compilation     | moderate within boundary |
| Slow schematic Gc | cross-context relational invariant | higher, if validated     |
| Gf re-entry       | flexible reuse under new demand    | far-transfer candidate   |

The revised model now distinguishes three transfer axes:

| Transfer axis           | Mechanism                            | Test                                                                                                       |
| ----------------------- | ------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| **Horizontal transfer** | invariant recovery across wrappers   | Does the same relation survive surface variation?                                                          |
| **Vertical transfer**   | cross-layer policy integration       | Does the same control policy link state, attention, WM/SR inference, prompt, action and delayed reuse?     |
| **Niche transfer**      | real-world cue and feedback coupling | Does the policy activate in the right environment, guide useful action, and generate informative feedback? |

# 3. State-space construction before SR inference

A key refinement is that not all training has to be SR training from the start.

There are two lower layers:

| Layer                    | Function                                       | Example                                                            |
| ------------------------ | ---------------------------------------------- | ------------------------------------------------------------------ |
| **Variable abstraction** | carve the state space into usable dimensions   | same/different, category boundary, relevant cue, variable identity |
| **SR inference**         | organise variables into predictive transitions | what follows, what changes, what action leads where                |

So association and discrimination training are useful when they help the learner distinguish:

```text
what matters
what is irrelevant
what variable is changing
what boundary defines the concept
```

Then SR training operates over those variables:

```text
given this state, what follows?
what action changes the state?
what future consequence should I expect?
```

This produces a two-tier theory:

> **Discrimination builds variables. SR training builds inference over variables.**

Niche coupling determines whether those variables are the right variables for the environment. A learner may carve a clean state space inside the task while still failing to identify the variables that matter in the real niche. Niche transfer therefore requires a further step:

task variable
→ wrapper-invariant variable
→ niche-valid variable

For example, in a puzzle, the relevant variable may be a rule, relation or path constraint. In a work or social setting, the relevant variable may be a role, incentive, boundary, cue, feedback signal, or affordance. Far transfer requires the learner to recognise when the trained variable has an analogue in the live niche.

In general intelligence training, this means moving from:

```text
surface cue → premature rule → brittle solution
```

to:

```text
surface cue → variable abstraction → relational inference → boundary test → portable schema
```

In resilience training, it means moving from:

```text
threat cue → fused appraisal → avoidance / over-control
```

to:

```text
threat cue → variable decomposition → alternative appraisal → prediction test → adaptive update
```

# 4. The global-workspace prompt layer

The meta-epistemic prompts occupy the **global-workspace / language-accessible control layer** of the protocol.

They do not train the reasoning operator directly. Instead, they broadcast a compact control policy that coordinates lower-level processes and makes them reusable across tasks.

Global workspace theory treats conscious access as involving selection and global availability of information for systems such as memory, planning, decision-making and report. This supports the idea that a compact verbal prompt can act as a broadcast token for a wider control policy, although it does not itself prove the Trident-G protocol. ([ScienceDirect][10])

A good prompt is therefore not an explanation. It is a **portable control handle**:

| Prompt                                | Lower-level operator    | Global-workspace function                       |
| ------------------------------------- | ----------------------- | ----------------------------------------------- |
| “What must be true?”                  | constraint satisfaction | broadcasts mutual-information constraint        |
| “What else could explain this?”       | counterfactual search   | broadcasts controlled entropy injection         |
| “What would make this wrong?”         | boundary testing        | broadcasts falsification / rule limitation      |
| “Which feature changed?”              | variable abstraction    | broadcasts relevant-variable selection          |
| “Can this path still reach the goal?” | graph/path SR search    | broadcasts look-ahead and reachability checking |

So:

```text
SR task trains the inference.
Puzzle or reappraisal mission trains the application.
Meta-epistemic prompt trains the thinker.
```

The vertical-transfer target is:

```text
implicit operator
→ language-accessible control prompt
→ action/test
→ outcome feedback
→ implementation cue
→ delayed reuse
```

# 5. Reinforcement of vertical transfer

Vertical transfer is strengthened when feedback does not merely reward final correctness, but reinforces successful **cross-layer coordination**.

The reinforced unit is:

```text
state readiness
→ attention-control stability
→ variable binding
→ SR-style inference
→ epistemic prompt selection
→ mission action
→ real-world implementation
→ delayed reuse
```

In reinforcement-learning terms, dopamine prediction-error signals provide a plausible biological mechanism for strengthening policies and cue–action relations when outcomes are better or worse than expected. Classic and later work on dopamine prediction error supports this general role in reward learning and temporal-difference style updating. ([PubMed Central][11])

In Trident-G terms, the key idea is **vertical credit assignment**:

```text
Did the learner succeed because attention was stable?
Because WM held the right variables?
Because the SR map supported the right transition prediction?
Because the epistemic prompt selected the right control policy?
Because the implementation cue fired in real life?
```

A good training system should therefore reinforce:

```text
accurate evidence sampling
correct variable selection
useful transition prediction
appropriate prompt use
successful boundary testing
implementation-cue use
delayed recovery of the same policy under a new wrapper
```

This prevents the protocol from becoming a shallow reward loop based only on speed or correctness. The target is not “get the answer right”. The target is:

```text
use the right adaptive control chain,
under the right state conditions,
across changed wrappers,
and later in real-world contexts.
```

# 6. Sensory discrimination and speed as controlled perturbations

Sensory discrimination contributes to far transfer only when it remains subordinate to the relational invariant.

Its role is:

> **to make the invariant recoverable from lower-quality, less redundant, less verbal evidence.**

It can help by:

| Sensory-discrimination role       | Cognitive function                                 |
| --------------------------------- | -------------------------------------------------- |
| raises input precision            | cleaner evidence for inference                     |
| forces attentional selection      | sample the relevant feature, not the obvious cue   |
| prevents semantic shortcutting    | less reliance on verbal/symbolic recoding          |
| improves prediction-error quality | errors become more informative                     |
| supports portability testing      | same relation must survive new perceptual wrappers |

Speed is a separate perturbation. It increases temporal throughput demand:

```text
detect → bind → compare → update → respond
```

Moderate speed pressure can prevent slow verbal recoding and test whether a relation has become efficient. Excessive speed collapses the task into processing-speed stress or guessing.

Therefore:

> **The relational variable should be the primary adaptive target. Sensory precision and speed should be secondary stressors used to test robustness once the relation is stable.**

# 7. Updated compact formulation

The stripped-down revised framework is:

> **Zone Coach trains and measures the state conditions for adaptive learning: Ψ-band access, attention-control stability, cognitive-control capacity and recovery from perturbation. WM–Puzzle Epistemic Transfer trains the G-structure: variable abstraction, SR-style reasoning operators, problem-space navigation, wrapper transfer and delayed deployment. WM–Reappraisal Cognitive Resilience trains the R-structure: flexible appraisal, threat-schema updating, prediction testing, recovery under load and real-life implementation. Meta-epistemic prompts provide the global-workspace control layer, converting implicit operator learning into language-accessible mindware scripts. Implementation intentions convert those scripts into cue-linked real-world action policies. Reinforcement signals strengthen the vertical chain when state, attention, WM/SR inference, prompt selection, mission action and delayed reuse successfully coordinate. Zhang–Tang entropy–mutual information dynamics provide the formal learning principle: productive transfer requires enough entropy to reconfigure and enough mutual information to remain relevant. Trident-G supplies the full adaptive-intelligence architecture: local SR tuning, breakpoint-triggered re-entry into the Ψ-band, horizontal portability validation, vertical control-policy integration and slow schematic Gc crystallisation.**

Or even more compactly:

```text
Zone trains the state.
G trains the structure.
R trains resilient reappraisal.
Prompts train the thinker.
Implementation intentions train deployment.
Reinforcement binds the verticals.
Wrapper swaps test the horizontals.
Delayed probes test slow Gc.
```

# 5. Applied protocol components and programme variants

The applied framework is now best understood as a set of **component protocols** rather than three single-purpose applications.

Each component trains a different layer of the Trident-G transfer stack. App products or programmes then combine these components in different proportions depending on the intended use case: adaptive intelligence training, cognitive readiness, mind–body readiness, performance under pressure, cognitive resilience, pain self-management support or cognitive longevity.

The components are:

| Component | Protocol                      | Primary function                                                                                                                                                                                                                                                                                                                                                                                                   |
| --------- | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Z**     | **Zone Track**                | Tracks cognitive readiness through an MFT-M adaptation, estimating cognitive bandwidth, attention-control state, environment/niche fit, routing needs and longitudinal readiness patterns                                                                                                                                                                                                                          |
| **Z-HRV** | **Zone–HRV Track**            | Extends Zone Track into a mind–body readiness protocol by combining the MFT-M adaptation with a unique RR-interval measurement method, producing a four-state ANS/stress-system classifier for recovery, activation, stabilisation and training-route decisions. It can also deliver state-contingent breathwork, meditation, recovery and lifestyle prompts to help improve the user’s in-zone baseline over time |
| **WM-G**  | **Working Memory g-Transfer** | Trains the active workspace for transfer through variable discrimination, abstraction, associative/relational n-back, WM binding and SR-style predictive inference                                                                                                                                                                                                                                                 |
| **PS**    | **Problem Space Inference**   | Trains efficient solution-path finding in classic portable puzzle-style problems, using constraint search, problem-space navigation, epistemic strategy cues, global-workspace control prompts, implementation intentions and real-world missions                                                                                                                                                                  |
| **R**     | **Appraisal Space Inference** | Trains cognitive resilience by treating emotionally loaded reactions as affective problem spaces: users infer hidden appraisal models, separate threat variables, choose discriminating probes, test predictions and update trigger → meaning → action → outcome relations                                                                                                                                         |

The important distinction is:

```text
PS = puzzle-based problem-space inference
R = emotionally loaded appraisal-space inference
````

Problem Space Inference is the IQ Coach puzzle layer. It trains efficient solution-path discovery in classic Tatham-style portable puzzle problems, where the learner must search a constrained problem space, identify the best next move, avoid premature commitment and use epistemic prompts to guide strategy.

Appraisal Space Inference is the Resilience Coach layer. It treats stress, anxiety, pain, setbacks, criticism and other emotionally salient events as affective inference problems. The user learns to map the trigger, body signal, prediction, meaning, evidence, action urge and outcome, then choose small discriminating probes that can update the underlying appraisal model.

So the architecture is modular:

```text
Z      = cognitive readiness and routing
Z-HRV  = mind–body readiness and regulation
WM-G   = active workspace, variable abstraction and SR-style predictive inference
PS     = puzzle-based problem-space inference
R      = appraisal-space inference under emotional or bodily load
```

## 5.1 Z: Zone Track

### Role in the framework

Zone Track is the **cognitive-readiness and state-routing protocol**.

It uses an MFT-M adaptation to estimate whether the learner is in a cognitive-control regime where deeper training is likely to work. The target is not a clinical diagnosis or a literal brain-state readout. It is a behavioural estimate of readiness, cognitive bandwidth and attention-control stability.

Zone Track asks:

```text
Am I cognitively ready for deeper training or adaptive action?
```

and:

```text
Is the current environment structured enough to support useful learning, feedback and deployment?
```

Its basic output can be:

```text
Zone state: Ready / Flat / Locked / Scattered
Niche fit: Clear / Opaque / Rigid / Chaotic
Route: Proceed / Reset / Activate / Clarify / Widen / Reduce entropy
```

### Primary function

Zone Track supports the rest of the stack by estimating:

```text
cognitive bandwidth
attention-control stability
response consistency
lapse / timeout pattern
state drift
environment/niche fit
route suitability
```

It is therefore a **gate and router**, not a complete training protocol by itself.

### How it combines with other components

| Combination   | Function                                                                      |
| ------------- | ----------------------------------------------------------------------------- |
| **Z + WM-G**  | Routes the user into full, reduced or stabilising working-memory/SR training  |
| **Z + PS**    | Decides whether the user is ready for puzzle-based problem-space inference    |
| **Z + R**     | Decides whether the user is ready for emotionally loaded appraisal-space work |
| **Z + Z-HRV** | Combines cognitive readiness with mind–body readiness                         |

### Primary evidence

The primary evidence for Z should be:

```text
MFT-M accuracy
response time
response-time variability
lapse / timeout rate
cognitive bandwidth estimate
state classification stability
state-label reliability across repeated sessions
prediction of next-block WM-G, PS or R performance
agreement with brief subjective readiness / niche-fit reports
```

The key validation question is:

```text
Does Zone Track route users into better training decisions than a fixed programme would?
```

### Claims boundary

Z can be framed as:

> A cognitive-readiness and training-route estimate based on an MFT-M adaptation.

It should not be framed as:

```text
a diagnosis of brain state
a clinical measure of anxiety, ADHD, fatigue or burnout
a guaranteed flow detector
a direct intelligence test
```

## 5.2 Z-HRV: Zone–HRV Track

### Role in the framework

Zone–HRV Track extends Zone Track into a **mind–body readiness protocol**.

It combines the MFT-M adaptation with a unique RR-interval measurement procedure to estimate patterns in the ANS/stress system and classify the user into a small set of actionable readiness/recovery regimes.

The four-state classifier should be treated as a **performance and wellbeing support model**, not a medical diagnostic system.

A simple operational structure is:

```text
MFT-M cognitive readiness
+
RR-interval / HRV-derived readiness signal
+
brief recovery / stress / sleep / context check
=
four-state readiness classification
```

The four states can be expressed commercially as something like:

```text
Ready / Stabilise / Recover / Activate
```

or technically as:

```text
regulated-ready
over-activated
under-activated
unstable / dysregulated
```

### Primary function

Z-HRV supports:

```text
mind–body readiness tracking
stress-system classification
training-route selection
recovery-route selection
baseline readiness improvement
state-contingent regulation prompts
```

It can deliver short interventions such as:

```text
paced breathing
longer-exhale breathing
brief grounding
short meditation
activation prompts
movement prompts
sleep / recovery prompts
lifestyle nudges
```

The important principle is that these are **state-contingent** rather than generic wellness tips:

```text
measure → classify → recommend regulation move → re-measure / track trend
```

### How it combines with other components

| Combination                   | Function                                                                                           |
| ----------------------------- | -------------------------------------------------------------------------------------------------- |
| **Z-HRV + WM-G**              | Decides whether the user should do full cognitive load, reduced load or recovery-oriented training |
| **Z-HRV + PS**                | Routes puzzle difficulty and pressure according to readiness                                       |
| **Z-HRV + R**                 | Stabilises the user before emotionally loaded appraisal-space work                                 |
| **Z-HRV + lifestyle prompts** | Builds a more stable in-zone baseline over time                                                    |

### Primary evidence

The primary evidence for Z-HRV should be:

```text
RR signal quality
artefact rate
usable recording percentage
within-session reliability
day-to-day repeatability
agreement with standard HRV-derived metrics
state-label stability
agreement with self-reported recovery / stress / readiness
prediction of training performance
prediction of recovery-route response
prediction of next-day readiness
```

The key validation question is:

```text
Does the combined MFT-M + RR-interval classifier improve readiness routing beyond MFT-M alone?
```

### Claims boundary

Z-HRV can be framed as:

> A mind–body readiness and recovery-support protocol using cognitive-readiness and RR-interval signals.

It should not be framed as:

```text
a medical-grade HRV assessment
a diagnosis of autonomic dysfunction
a treatment for stress, anxiety, burnout or cardiovascular conditions
a replacement for medical or psychological care
```

## 5.3 WM-G: Working Memory g-Transfer

### Role in the framework

Working Memory g-Transfer is the **active workspace and predictive-relation training protocol**.

It trains the ability to hold, update and compare variables under controlled load. It does this through:

```text
variable discrimination
variable abstraction
associative n-back
relational n-back
working-memory binding
SR-style predictive inference
```

Its role is not to claim that ordinary working-memory training directly increases intelligence. Its role is to train the active workspace in which relational variables, appraisals, counterexamples, constraints and predictions can be held and updated while the rest of the protocol tests portability.

### Primary function

WM-G trains:

```text
what matters versus what is irrelevant
which feature changed
which relation is preserved
which item predicts which future state
which variable must be held online
which lure should be inhibited
which relation survives a wrapper change
```

It therefore supports both adaptive-intelligence training and resilience training.

For IQ Coach, it provides the run-time cognitive substrate for reasoning.

For Resilience Coach, it supports the ability to hold:

```text
trigger → prediction → alternative model → small test → outcome
```

without collapsing back into the fused threat interpretation.

### Relationship to SR theory

WM-G is inspired by the idea that transfer depends on learning predictive relations, not only item-response mappings.

The mechanistic hypothesis is:

```text
state → transition → consequence → next reachable state
```

rather than:

```text
item → response
```

The hippocampus and parahippocampal system are relevant theoretical anchors because of their role in relational memory, predictive maps and successor representations. However, app-level claims should remain behavioural unless neural validation is conducted.

### How it combines with other components

| Combination      | Function                                                                              |
| ---------------- | ------------------------------------------------------------------------------------- |
| **WM-G + Z**     | Cognitive load is adjusted according to readiness                                     |
| **WM-G + PS**    | Variable binding and relation tracking support efficient puzzle solution paths        |
| **WM-G + R**     | Users hold appraisal variables and alternative predictive models under emotional load |




## 5.4 PS: Problem Space Inference

### Role in the framework

Problem Space Inference is the **puzzle-based adaptive-intelligence protocol**.

It trains efficient solution-path finding in classic portable puzzle-style problems. These problems are inspired by the logic of small, self-contained puzzle systems such as Tatham-style portable puzzles: bounded rules, generated instances, clear constraints, local deductions and efficient solution paths.

The key target is:

```text
current state → possible moves → constraints → best next move → updated state → solution path
```

This is not the same as Appraisal Space Inference. PS is the IQ Coach puzzle protocol. It trains abstract and semi-abstract problem-space navigation.

### Primary function

PS trains:

```text
constraint search
deductive propagation
probe selection
hypothesis updating
best-next-move selection
path finding
re-representation
strategy switching
efficient solution routes
```

The embedded prompts train the global-workspace strategy layer:

```text
What must be true?
What is the best next probe?
Can this path still reach the goal?
What would make this rule wrong?
Which move reduces uncertainty most?
```

These prompts should become reusable **mindware scripts**.

### General item/session structure

A PS session can use classic portable puzzle-style formats such as:

```text
grid constraints
path constraints
visibility / skyline constraints
hidden-object inference
loop/path construction
region constraints
logic placement
candidate elimination
```

The app does not need to explain the formal operator. The puzzle makes the learner perform it.

The central loop is:

```text
1. Present a bounded puzzle state.
2. Ask for the best next move, probe or inference.
3. Provide feedback on why the move was efficient or inefficient.
4. Prompt the relevant mindware script.
5. Vary the wrapper or puzzle family.
6. Test whether the same strategy survives changed surfaces.
7. Link the strategy to a real-world mission when appropriate.
```

### How it combines with other components

| Combination       | Function                                                                                                 |
| ----------------- | -------------------------------------------------------------------------------------------------------- |
| **PS + Z**        | Puzzle difficulty and route are adjusted according to cognitive readiness                                |
| **PS + WM-G**     | Working-memory/SR training supports relation tracking and efficient puzzle search                        |
| **PS + R**        | PS can provide general cognitive flexibility, but should not be treated as the core resilience mechanism |
| **PS + missions** | Puzzle strategies are translated into real-world “best next move” tasks                                  |

### Important distinction for resilience applications

For chronic pain or emotional resilience, PS should not be treated as the direct intervention if PS means Tatham-style puzzle training.

In those cases:

```text
PS = optional general cognitive-flexibility support
R = direct appraisal-space intervention
```

A pain-support or anxiety-support track may borrow the language of “best next probe” or “best next move”, but the direct emotional-update mechanism belongs to R, not to classic puzzle PS.

### Primary evidence

The primary evidence for PS should be:

```text
completion time
move count
undo count
hint use
error count
time to first useful move
forced-move detection latency
branching-factor reduction
candidate elimination accuracy
probe efficiency
hypothesis update accuracy
post-error recovery
performance after wrapper swap
delayed retention
real-world mission completion
```

The key validation question is:

```text
Do users become more efficient at finding solution paths across changed puzzle wrappers and later apply the same mindware scripts in real-world missions?
```

### Claims boundary

PS can be framed as:

> Puzzle-based problem-space inference training for efficient solution-path finding and strategy transfer.

It should not be framed as:

```text
guaranteed real-world decision improvement
direct treatment for emotional or clinical problems
proof of IQ increase by itself
automatic transfer to all problem-solving contexts
```

## 5.5 R: Appraisal Space Inference

### Role in the framework

Appraisal Space Inference is the **cognitive-resilience protocol**.

It treats emotionally loaded reactions as affective problem spaces. Instead of only asking the user to challenge an automatic thought, it trains the user to infer and update a predictive appraisal model.

The core structure is:

```text
trigger
→ body signal
→ emotion
→ prediction
→ meaning
→ action urge
→ action/probe
→ outcome
→ updated model
```

The central question is:

```text
Can the learner hold a trigger, body signal, prediction, meaning, evidence, action urge and small test in working memory long enough to update the underlying schema?
```

This is not merely positive thinking. It is **predictive reappraisal**.

### Primary function

R trains:

```text
appraisal discrimination
threat-variable separation
hidden appraisal model inference
alternative model generation
evidence selection
best-probe selection
prediction testing
implementation intention formation
real-world outcome logging
delayed re-use of the new relation
```

Its core transformation is:

```text
trigger → fused threat interpretation → maladaptive action
```

becomes:

```text
trigger → appraisal discrimination → alternative predictive model → small probe → outcome feedback → schema update
```

### Appraisal-space variables

The user learns to distinguish:

```text
trigger
sensation
emotion
prediction
meaning
evidence
action urge
chosen action
outcome
recovery signal
```

This gives R its own version of the state-space construction principle:

```text
Appraisal discrimination builds variables.
Reappraisal-SR training builds adaptive predictive relations.
```

Examples:

| Domain         | Old predictive map                    | New adaptive map                                                      |
| -------------- | ------------------------------------- | --------------------------------------------------------------------- |
| Pain           | sensation → damage → avoidance        | sensation → information/context → graded action → recovery/confidence |
| Exam pressure  | anxiety → failure → freezing          | anxiety → preparation cue → start routine → performance feedback      |
| Sport pressure | arousal → danger → overcontrol        | arousal → readiness signal → routine → first action                   |
| Work criticism | criticism → rejection → defensiveness | feedback → information → revision → competence                        |
| Perfectionism  | error → inadequacy → overcontrol      | error → information → correction → progress                           |

### Difference from CBT-style restructuring

R can include familiar cognitive-restructuring elements, but it should not be reduced to a digital thought record.

A traditional surface form is:

```text
situation → automatic thought → evidence for/against → balanced thought
```

The Trident-G R form is:

```text
trigger → appraisal model → predicted transition → action tendency → outcome probe → schema update
```

So the target is not only a more balanced sentence. The target is a new predictive relation that can survive:

```text
emotional load
changed real-world cues
delay
future recurrence
```

### How it combines with other components

| Combination   | Function                                                                                                  |
| ------------- | --------------------------------------------------------------------------------------------------------- |
| **R + Z**     | Cognitive readiness determines whether to proceed, simplify or stabilise                                  |
| **R + Z-HRV** | Mind–body regulation supports safe engagement with emotionally loaded material                            |
| **R + WM-G**  | WM-G supports holding the appraisal chain and alternative model online                                    |
| **R + PS**    | PS may support general problem-solving flexibility, but R remains the direct affective inference protocol |

### Primary evidence

The primary evidence for R should be:

```text
trigger identification
emotion/sensation separation
belief/prediction identification
evidence selection
alternative-model generation
best-probe choice
threat-lure errors
action-urge recognition
implementation intention creation
real-world cue detection
real-world action/probe completion
outcome tracking
recovery latency
delayed re-use
```

The key validation question is:

```text
Do users become better at inferring and updating threat-loaded predictive maps under real-world cues, rather than only completing in-app reflection items?
```

### Claims boundary

R can be framed as:

> Cognitive resilience and appraisal-space inference practice.

It should not be framed as:

```text
treatment for anxiety disorder
treatment for depression
treatment for PTSD
treatment for chronic pain
treatment for ADHD
a replacement for CBT or clinical care
a digital therapeutic
```

without separate clinical validation, professional oversight and appropriate regulatory review.

# 8. Entropy–mutual information model across the components

The computational model can treat learning as a sequence of updates to a cognitive, puzzle or appraisal map.

At each trial, move, inference or reappraisal exercise:

```text
H_t = entropy over possible variables, hypotheses, moves, meanings or appraisals
I_t = mutual information between the selected update and task-relevant outcome
Δ_t = update magnitude to the map or policy
β_t = directional alignment of recent updates
```

The productive regime is:

```text
sufficient entropy to explore alternatives
+
sufficient mutual information to remain relevant
+
occasional large updates
+
stable small updates
```

## 8.1 Z and Z-HRV

Z manipulates and measures cognitive-control demand:

```text
stimulus entropy
masking difficulty
response ambiguity
feature uncertainty
speed / exposure time
```

Z-HRV adds a mind–body readiness layer:

```text
RR-interval pattern
ANS/stress-system state
recovery/activation regime
readiness-route fit
```

The target is:

```text
task challenge high enough to require control
+
response behaviour informative enough to support learning
+
mind–body state stable enough to sustain training
```

## 8.2 WM-G

In WM-G:

```text
entropy = possible variables, relations, bindings or lures
mutual information = how much the selected variable/relation predicts the correct update
```

The learner improves when they can identify and hold the right variable or relation across changed wrappers.

## 8.3 PS

In PS:

```text
entropy = possible moves, paths, hypotheses or puzzle states
mutual information = how much a move reduces uncertainty about the solution
```

A good learner does not simply move quickly. They select moves that reduce the problem space efficiently.

## 8.4 R

In R:

```text
entropy = possible meanings, explanations, predictions and actions
mutual information = fit between appraisal, evidence, action and outcome data
```

A maladaptive appraisal often has low entropy and high subjective certainty:

```text
pain means damage
anxiety means failure
criticism means rejection
mistake means inadequacy
```

R introduces controlled entropy:

```text
What else could this mean?
What evidence distinguishes these models?
What small test would update this?
```

Then it restores mutual-information constraint:

```text
prediction → action/probe → outcome → recovery → belief update
```

# 10. Validation philosophy

This framework should be presented as an **evidence-generating theory**, not as a proven far-transfer, clinical or medical intervention.

The claims ladder is:

| Evidence level                  | Claim supported                                                                                 |
| ------------------------------- | ----------------------------------------------------------------------------------------------- |
| Theoretical/computational       | Model predicts transfer under specified conditions                                              |
| Measurement reliability         | The app measures a signal consistently                                                          |
| App telemetry                   | Users improve on trained and structurally related tasks                                         |
| Variable-abstraction evidence   | Users increasingly identify relevant variables and suppress lures                               |
| Horizontal portability evidence | Gains survive wrapper swaps, boundary probes and delayed re-checks                              |
| Vertical-transfer evidence      | State, WM/SR, prompt use, implementation cue use and delayed deployment form a predictive chain |
| Niche-transfer evidence         | Skills are deployed under real-world cues with outcome feedback                                 |
| Active-control evidence         | Protocol outperforms matched alternatives                                                       |
| Applied evidence                | Benefits appear in target settings such as study, work, sport or healthy ageing                 |
| Clinical evidence               | Only after formal trials, safety review and regulatory consideration                            |

## 10.1 General claim boundary

The whole framework can claim:

> These protocols train and track cognitive readiness, working-memory binding, problem-space inference, appraisal-space inference and real-world deployment under specified conditions.

```text
proven IQ increase
clinical treatment
medical diagnosis
guaranteed far transfer
guaranteed academic, work, sport or health outcomes
```

without controlled evidence.

```text
Z tracks readiness.
```
