# Trident-G Theory Summary

## 1. Core Trident-G theory

**Trident-G defines general adaptive intelligence as the capacity to stay viable in a near-critical cognitive regime, search relational problem spaces without collapsing into rigidity or scatter, validate useful structures across variation, and crystallise what survives into reusable Gc.**

Adaptive intelligence involves maintaining near-critical cognition, searching relational spaces, validating structures across variation, and crystallising what survives into reusable Gc for later flexible re-entry under Gf demand.

The **G-loop** is not a simple step-by-step procedure. It is a concurrent, multi-layer dynamical system coupled through α-stability monitoring, hippocampal successor representation maps, fast and slow Gc construction pathways, and allostatic regulation through the Ψ-band. Each loop iteration should become a **spiral**, not a mere repetition: the system returns to later problems with a richer representational starting point.

The central dynamical claim is that adaptive intelligence depends on a balance between two forces:

| Force                               | Function                                                              | Failure if dominant                   |
| ----------------------------------- | --------------------------------------------------------------------- | ------------------------------------- |
| **Entropy / exploration**           | opens the search space, generates candidate relational configurations | scattered exploration, no convergence |
| **Mutual information / constraint** | filters search by relevance, goal, outcome and prediction error       | premature closure, brittle Gc         |

Layer 1 of the theory defines this as a sustained bifurcating dynamic: FPCN-B supports broad SR-structured relational search, while FPCN-A applies goal-relevant constraint. The system stays adaptive when neither dominates long enough to collapse the dynamic.

Trident-G also includes **niche coupling**. Intelligence is not only an internal near-critical state. It depends on whether the organism’s internal search space remains coupled to the external world of affordances, cues, constraints, feedback loops, risks, rewards and action consequences.

So the basic Trident-G formula is:

```text
adaptive intelligence
= Ψ-band access
+ entropy–MI balance
+ relational map search
+ niche coupling
+ portability validation
+ Gc crystallisation
+ flexible re-entry under new demand
````

## 2. The Ψ-band

The **Ψ-band** is the operational corridor in which the system is neither too rigid nor too fragmented. It is the zone where Gf reconfiguration can occur without falling into:

```text
subcritical rigidity = fast but narrow automation
supercritical fragmentation = broad but unstable exploration
```

In computational terms, Trident-G defines **g** as an increasing function of two things: time spent in the Ψ-band under challenge, and the volume/diversity of G-loops the organism can reliably enter and re-enter.

That is an important shift. Intelligence is not just “how many problems can you solve?” It is closer to:

```text
How much near-critical adaptive search can you sustain?
Across how many kinds of relational trajectory?
While remaining coupled to the real niche?
Without collapsing into rigidity, scatter, overload or decoupled fantasy?
```

## 3. Variable abstraction comes before SR inference

A key refinement in the theory is that SR-style inference does not operate directly over raw surfaces. First, the learner/system has to carve the state space into usable variables.

The core sequence is:

```text
surface cue
→ variable abstraction
→ SR inference
→ boundary test
→ portable schema
```

rather than:

```text
surface cue
→ premature rule
→ brittle solution
```

Association and discrimination are therefore not “low-level extras”. They help identify the variables over which later SR inference can operate.

So the basic learning hierarchy is:

```text
discrimination builds variables
WM binds variables
SR training learns transitions over variables
boundary testing validates the relation
slow Gc extracts the invariant
```

This is one of the most important clarifications, because it prevents the theory from jumping too quickly from “task practice” to “relational transfer”. The system first has to know what the relevant dimensions are.

## 4. Successor representation is the relational map mechanism

The SR layer is where the system learns predictive structure:

```text
state
→ transition
→ consequence
→ next reachable state
```

rather than merely:

```text
stimulus
→ response
```

Horizontal SR tuning means improving the predictive map across changed wrappers, feature instantiations and contexts. This is initially efficient exploitation within a current relational structure. But the deeper mechanism comes when the system probes whether a longer temporal horizon reveals a deeper structure before performance fully plateaus.

So the SR layer does two jobs:

1. **Local tuning**: getting better within a current relational map.
2. **Horizon extension**: discovering whether a deeper, more general map explains the current one.

That second function is where the theory becomes a far-transfer theory rather than a practice-effects theory.

## 5. Fast Gc versus slow schematic Gc

Trident-G distinguishes two Gc construction pathways.

**Fast Gc** is compiled, efficient, and often task/procedure-like. It occurs when a relational configuration has been validated enough that uncertainty is low and the bifurcating dynamic resolves into subcritical SR automation. It is useful, but can become narrow or brittle if banked too early.

**Slow schematic Gc** is the deeper target. It is not just a stored strategy. It is a change in representational geometry. Across episodes, domains, wrappers, niches and loop iterations, the slow pathway extracts recurring invariants and installs them into cortical structure through consolidation, especially rest and sleep.

This gives the key distinction:

```text
surface skill = task-specific improvement
fast Gc = efficient but bounded strategy
slow schematic Gc = cross-context invariant geometry
Gf re-entry = flexible reuse under new demand
```

Without consolidation, the spiral property fails. Without adequate consolidation, schematic geometry is not installed and the loop degrades into flat cycling rather than self-enriching spiralling.

## 6. Horizontal transfer

**Horizontal transfer** means the same relational invariant survives surface variation. It is the recovery of the same invariant across changed wrapper, boundary case, novel context and delayed re-check.

So the horizontal-transfer test is not:

```text
Did performance improve on the trained task?
```

It is:

```text
Can the same structure be recovered when the surface changes?
Can it survive boundary cases?
Can it survive a novel context?
Can it survive after delay?
```

This is the anti-Lumosity principle: do not trust trained-task gain unless it survives wrapper variation and delayed re-check.

## 7. Vertical transfer

**Vertical transfer** means the same adaptive control policy can be reconstructed across layers of the system:

```text
Ψ-band access
→ controlled attention
→ WM variable binding
→ SR-style transition inference
→ global-workspace control policy
→ action selection
→ real-world feedback
→ delayed reuse
```

Vertical transfer is not merely knowing a rule. A vertically integrated policy can be entered from the right state, held in WM, used to guide SR search, broadcast as a control policy, deployed under real-world cue conditions, and later recovered after delay.

This separates:

```text
I can solve this format
```

from:

```text
I can re-enter the right cognitive state,
select the right variables,
run the right inference,
use the right control policy,
act on it,
learn from feedback,
and reuse it later.
```

## 8. Niche transfer

**Niche transfer** asks whether a policy remains usable in the real environment.

Horizontal transfer asks whether the structure survives changed task wrappers. Vertical transfer asks whether the policy reconstructs across cognitive layers. Niche transfer asks whether that policy remains coupled to the real environment.

A skill has not fully transferred merely because it survives a changed task format or can be reconstructed through a vertical control chain. It must also remain coupled to:

```text
real-world cues
affordances
constraints
feedback loops
social meanings
risks
rewards
action consequences
```

So niche transfer asks:

```text
Does the policy activate in the right environment?
Does it guide useful action?
Does the world provide informative feedback?
Can the learner update from that feedback?
Can the policy be reused later under changed niche conditions?
```

This matters because a learner can perform well inside a training task while failing to recognise the relevant cue in ordinary life. In Trident-G terms, this is low λ: the internal G-loop is not sufficiently coupled to the external niche.

## 9. Breakpoint-driven transfer

The transfer protocol rests on a breakpoint principle.

Ordinary training often produces smooth local gains and then fast task-specific automation. The far-transfer protocol instead aims to perturb the learner near local asymptote, before surface-specific automation hardens.

The cycle is:

```text
hold the invariant constant
→ vary wrappers
→ detect local asymptote
→ perturb the surface while preserving the invariant
→ look for dip-and-recovery
→ validate portability
→ probe niche coupling
→ re-test after delay
→ treat the structure as a slow schematic Gc candidate
```

The ideal learning signature is:

```text
local tuning
→ asymptote
→ controlled perturbation
→ temporary dip
→ faster recovery
→ improved portability
→ niche-valid deployment
→ delayed survival
```

The dip matters. It suggests the learner is not merely repeating a surface routine. They are being forced to re-enter the Ψ-band and recover the deeper invariant.

## 10. Prompt/control-policy layer

Some G-loop operations become language-accessible when a lower-level relational operation is broadcast as a compact control policy.

Examples include:

```text
What must be true?
What else could explain this?
What would make this wrong?
Which feature changed?
Can this path still reach the goal?
```

The important point is that a prompt is not the intelligence itself. It is a **control handle**. It helps coordinate attention, WM binding, SR search, prefrontal constraint and action.

So a “mindware script” is best understood as:

```text
compact verbal handle
→ selects an epistemic stance
→ coordinates lower-level processes
→ supports reuse across wrappers, contexts and niches
```

## 11. The clean transfer principle

The transfer framework’s core thesis is:

```text
Far transfer is not produced by repeating a task.
It is produced by:

1. maintaining trainable Ψ-band access,
2. constructing the right variables,
3. tuning predictive relational maps,
4. perturbing wrappers near asymptote,
5. forcing controlled re-entry before brittle automation,
6. validating portability across boundary cases, context and delay,
7. integrating the policy vertically across state, attention, WM, SR, action and feedback,
8. coupling the policy to real-world niche cues, affordances and consequences,
9. allowing consolidation into slow schematic Gc.
```

Far transfer therefore requires near-critical regime maintenance, predictive relational map tuning across surface variation, controlled breakpoint transitions, portability validation, cross-layer control-policy strengthening, niche-coupled feedback, and consolidation into slow schematic Gc.

## 12. What the theory should not claim yet

The strongest version of the theory is **evidence-generating**, not already proven as a complete intervention.

There is evidence for components such as strategy instruction, self-explanation, metacognitive prompting, strategy-enriched training, inductive reasoning training and implementation intentions. But the full Trident-G transfer stack remains an evidence-generating hypothesis.

So the clean claims boundary is:

```text
Defensible:
This is a mechanistic theory of how far transfer could be engineered and tested.

Not yet defensible:
This protocol has already been proven to raise g in the strong psychometric sense.
```

## Bottom line

At the most basic level:

```text
Trident-G = a theory of adaptive intelligence as Ψ-band navigation, niche coupling and spiral advance.

Transfer protocol = a method for forcing learning out of surface-specific fast Gc and into validated, consolidated, reusable slow schematic Gc.

Horizontal transfer = invariant survives changed wrappers.

Vertical transfer = control policy reconstructs across state, attention, WM, SR inference, action, feedback and delayed reuse.

Niche transfer = policy remains coupled to real-world cues, affordances, constraints, feedback and consequences.

Far transfer = horizontal portability + vertical integration + niche coupling + consolidation.
```

That is the clean theoretical core before bringing the specific app protocols back in.
